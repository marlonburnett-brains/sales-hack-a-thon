### Phase 22 Gap Detail

The `GoogleTokenBadge` component reads the `google-token-status` cookie via `document.cookie`, but the cookie is set with `httpOnly: true` in both `middleware.ts` and `auth/callback/route.ts`. HttpOnly cookies are inaccessible from client-side JavaScript, so:
- Badge always shows for ALL users (including those with valid tokens)
- Badge never disappears after successful consent

**Impact:** UI indicator only. Core OAuth flow, token storage, and encryption are fully functional. No requirements are blocked.

**Fix:** Remove `httpOnly` from the `google-token-status` cookie (it contains only "valid"/"missing" strings, not tokens), OR change the badge to receive token status as a server component prop.

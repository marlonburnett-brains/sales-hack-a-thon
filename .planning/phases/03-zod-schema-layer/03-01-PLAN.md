<objective>
Define all Zod v4 schemas, domain constants, and the zodToGeminiSchema helper in packages/schemas as a shared package.

Purpose: Establish the complete schema layer that all downstream phases (4-11) import from. Every LLM schema is flat/Gemini-safe (no transforms, no optionals, no unions). Every schema exports both a Zod schema and an inferred TypeScript type. Domain constants are consolidated from Phase 2's classify-metadata.ts into a single source of truth.

Output: 13 schema files (10 LLM + 2 app + 1 consolidated from Phase 2), 1 constants file, 1 helper utility, and 1 barrel export index.

    - Returns a plain `Record<string, unknown>` JSON Schema object
    - Inherits z.toJSONSchema()'s throw behavior on unsupported features (transforms, maps, sets, symbols) — this satisfies CONTEXT.md's "fail fast" requirement
    - NO coupling to `@google/genai` library — returns raw JSON Schema only

    Schema definitions (Claude's discretion on exact field names/nesting, guided by REQUIREMENTS.md and CONTEXT.md):

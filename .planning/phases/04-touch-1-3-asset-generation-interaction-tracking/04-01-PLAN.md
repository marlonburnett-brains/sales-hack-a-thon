    5. Create assembleFromTemplate(params: { templateId, targetFolderId, deckName, textReplacements: Record<string, string>, imageReplacements?: Record<string, string> }) that:
       - Copies template to target folder via Drive files.copy with supportsAllDrives: true
       - Builds batchUpdate requests array: replaceAllText for each text replacement (matchCase: true), replaceAllShapesWithImage for each image replacement (CENTER_INSIDE, matchCase: true)
       - Executes batchUpdate if requests.length > 0
       - Calls makePubliclyViewable on the new presentation
       - Returns { presentationId, driveUrl: `https://docs.google.com/presentation/d/${presentationId}/edit` }
       - This is the REUSABLE module that Phase 8 will extend. Keep it generic -- no Touch-specific logic.

    **Touch 1 Mastra workflow (apps/agent/src/mastra/workflows/touch-1-workflow.ts):**

    6. Create workflow with these steps chained via .then():
       - **generateContent**: Receives { dealId, companyName, industry, context, salespersonName }. Calls Gemini 2.5 Flash with PagerContentLlmSchema via zodToGeminiSchema. Returns generated PagerContent.
       - **awaitApproval**: Uses Mastra suspend/resume pattern. Suspends with the generated content so the web app can display it. Resume schema accepts { decision: 'approved' | 'edited', editedContent?: PagerContent }. Returns { finalContent, decision }.
       - **assembleDeck**: Calls assembleFromTemplate with the Touch 1 pager template. Text replacements: `{{company-name}}` -> companyName, `{{headline}}` -> headline, `{{value-proposition}}` -> valueProposition, `{{capabilities}}` -> keyCapabilities.join(', '), `{{call-to-action}}` -> callToAction. Returns { presentationId, driveUrl }.
       - **recordInteraction**: Creates InteractionRecord in Prisma with status 'approved' or 'edited', creates FeedbackSignal (positive for approved, negative for edited with diff content), ingests the generated deck into AtlusAI via ingestDocument. Returns the interaction record.

    7. Register touch-1-workflow in apps/agent/src/mastra/index.ts by importing and adding to the workflows object.

    8. Add a custom API route on the Mastra server for file upload (Touch 1 override): POST /api/touch-1/upload that accepts a Google Slides file upload, copies it to the deal's Drive folder, creates an InteractionRecord with decision 'overridden', creates a FeedbackSignal with signalType 'override', and ingests the uploaded file into AtlusAI. Use Mastra's registerApiRoute pattern (Hono-based). Alternatively, if registerApiRoute is not straightforward, create this as a separate endpoint in the workflow or as a standalone Hono route.

    **Environment:**

    9. Ensure GEMINI_API_KEY is required (not optional) in apps/agent/src/env.ts since Touch 1 workflow needs Gemini for content generation. Change from `.optional()` to `.min(1)`.

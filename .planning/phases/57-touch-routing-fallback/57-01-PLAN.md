  <action>
    Create `apps/agent/src/generation/route-strategy.ts` with three exported functions:

    1. **`resolveGenerationStrategy`** -- Three-way routing decision:
       - Takes `touchType: string`, `artifactType: ArtifactType | null`, `dealContext: DealContext`
       - Calls `resolveBlueprint({ touchType, artifactType }, dealContext)` -- if null, return `{ type: "legacy" }`
       - Queries `prisma.deckStructure.findFirst({ where: { touchType, artifactType }, select: { exampleCount: true } })` for confidence check
       - Calls `calculateConfidence(exampleCount)` -- if green, return `{ type: "structure-driven", blueprint, confidence }`; else return `{ type: "low-confidence", blueprint, confidence }`
       - Export discriminated union type `GenerationStrategy` with three variants

    2. **`buildDealContext`** -- Constructs DealContext from touch workflow inputs:
       - Takes `touchType: string` and `input: { dealId: string; companyName: string; industry: string; [key: string]: unknown }`
       - Maps touchType to funnelStage: touch_1="First Contact", touch_2="Intro Conversation", touch_3="Capability Alignment", touch_4="Solution Proposal"
       - Uses `input.capabilityAreas` as pillars if present, else empty array
       - Uses `input.priorTouchOutputs` as priorTouchSlideIds if present, else empty array
       - persona defaults to "General"

    3. **`executeStructureDrivenPipeline`** -- Full pipeline orchestration:
       - Takes `{ blueprint: BlueprintWithCandidates; targetFolderId: string; deckName: string; dealContext: DealContext; ownerEmail?: string }`
       - Returns `Promise<AssembleDeckResult>`
       - Step 1: `selectSlidesForBlueprint(blueprint)` to get `SectionMatchResult` with `plan: SlideSelectionPlan`
       - Step 2: Query all slides per involved presentation from `SlideEmbedding` table (grouped by templateId -> Template.presentationId). Get unique presentationIds from `plan.selections`, query Template for presentationId mapping, then query SlideEmbedding grouped by presentation. Build `Map<string, string[]>`.
       - Step 3: `buildMultiSourcePlan(plan, allSlidesByPresentation)` to get `MultiSourcePlan`
       - Step 4: `assembleMultiSourceDeck({ plan: multiSourcePlan, targetFolderId, deckName, ownerEmail })` to get `AssembleDeckResult`
       - Step 5: Plan modifications -- for each selection in the plan, call `planSlideModifications({ slideId: selection.slideId, slideObjectId: selection.slideId, dealContext })`. Use the slideId as slideObjectId because for primary slides the objectId is preserved, and for secondary slides the assembler maps them.
       - Step 6: Filter out fallback/empty plans, call `executeModifications({ presentationId: assemblyResult.presentationId, plans: activePlans })` if any active plans exist
       - Return `assemblyResult`

    The `getAllSlidesByPresentation` helper is a private async function inside route-strategy.ts. It takes an array of presentationIds, queries Template to get templateId->presentationId mapping, then queries SlideEmbedding to get all non-archived slides grouped by presentation.

    **Important:** Import `prisma` from `../lib/db`. Import all pipeline functions from their respective modules. Import `calculateConfidence` from `../deck-intelligence/deck-structure-schema`. Import `type ArtifactType` from `@lumenalta/schemas` (or from deck-structure-key if that's where ArtifactType is defined -- check the import used in deck-structure-key.ts).

    - Test `resolveGenerationStrategy` returns `{ type: "low-confidence" }` when confidence is yellow/red
    - Test `buildDealContext` constructs correct DealContext for each touch type with proper defaults
    - Do NOT test `executeStructureDrivenPipeline` (requires too many mocks for unit tests -- integration tested via workflow execution)

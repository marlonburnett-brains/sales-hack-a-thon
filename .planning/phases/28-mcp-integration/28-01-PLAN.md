**Create `apps/agent/src/lib/mcp-client.ts`** -- the MCPClient singleton wrapper.

Module-level state:
- `mcpClient: MCPClient | null` -- the current instance
- `createdAt: number` -- when the current instance was created
- `fallbackMode: boolean` -- set true if MCP unreachable at boot, prevents repeated connection attempts
- `initializing: Promise<void> | null` -- prevents concurrent initialization

**`createMCPClient()` (internal):**
- Creates a new `MCPClient` from `@mastra/mcp` with config:
  - `id: "atlus-ai"`
  - `servers: { "atlus-ai": { url: new URL("https://knowledge-base-api.lumenalta.com/sse"), fetch: authFetch, timeout: 15_000 } }`
  - `timeout: 15_000`
- The `fetch` callback (`authFetch`): calls `getPooledAtlusAuth()` from `atlus-auth.ts`, throws if no token available, injects `Authorization: Bearer {token}` header into the request. This ensures each HTTP request gets a fresh token from the pool (MCP-04).

**`initMCPClient()` (exported):**
- Called once at agent boot (eager connection per user decision)
- If `env.ATLUS_USE_MCP` is 'false', log and return immediately
- Creates the MCPClient via `createMCPClient()`
- Runs health probe: `await mcpClient.listTools()` with a 5-second timeout wrapper
- On success: log tool count, set `fallbackMode = false`
- On failure: log warning, set `fallbackMode = true`, disconnect the client, set `mcpClient = null`
- Do NOT throw -- agent must boot even if MCP is down

**`getMCPClient()` (exported):**
- Returns `MCPClient | null`
- Returns null if: `env.ATLUS_USE_MCP === 'false'`, or `fallbackMode === true`
- Lazy recycle check (MCP-05): if `mcpClient` exists and `Date.now() - createdAt > maxLifetime`, disconnect the old client (catch errors), set `mcpClient = null`, create a new one via `createMCPClient()`, run health probe, update `createdAt`
- If `mcpClient` is null and not in fallback mode, create and probe (handles case where recycling created a null state)
- Returns the mcpClient instance or null

**`shutdownMCPClient()` (exported):**
- Calls `mcpClient?.disconnect()` in a try/catch, logs result
- Sets `mcpClient = null`

**SIGTERM handler** (registered at module scope):
- `process.on("SIGTERM", async () => { await shutdownMCPClient(); })` -- do NOT call `process.exit()` (let Mastra's own handler finish)

**Wire into agent boot in `apps/agent/src/mastra/index.ts`:**
- Import `initMCPClient` and `shutdownMCPClient` from `../lib/mcp-client`
- In the startup section (after staleness polling setup, before the export), add a fire-and-forget call: `initMCPClient().catch(err => console.error("[mcp] Init failed:", err));`
- The SIGTERM handler is already registered at module scope in mcp-client.ts, no need to add another one in index.ts

**Anti-patterns to avoid (from research):**
- Do NOT create MCPClient per request (SSE connections are expensive)
- Do NOT block startup on MCP connection (fire-and-forget init)
- Do NOT use background timers for recycling (lazy check before each request per user decision)
- Do NOT import MCP in apps/web (MCP-02)

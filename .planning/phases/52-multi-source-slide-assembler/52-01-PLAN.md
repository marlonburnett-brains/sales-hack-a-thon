  <name>Multi-Source Plan Builder</name>
  <files>apps/agent/src/generation/multi-source-assembler.ts, apps/agent/src/generation/__tests__/multi-source-assembler.test.ts</files>
  <behavior>
    groupSlidesBySource:
    - Given selections from 3 sources (A:3, B:2, C:1), returns Map with 3 entries keyed by presentationId
    - Given selections all from one source, returns Map with 1 entry
    - Given empty selections, returns empty Map

    identifyPrimarySource (internal, tested via buildMultiSourcePlan):
    - Given groups {A:5, B:2, C:1}, identifies A as primary
    - Given tie {A:3, B:3}, picks the first one encountered (deterministic via insertion order)

    buildMultiSourcePlan:
    - Given selections from 2 sources (A:3 slides [s1,s2,s3], B:1 slide [s4]) and allSlidesByPresentation map:
      - primarySource.presentationId = A
      - primarySource.keepSlideIds = [s1,s2,s3]
      - primarySource.deleteSlideIds = all A slides NOT in [s1,s2,s3]
      - secondarySources = [{presentationId: B, slideIds: [s4]}]
      - finalSlideOrder = [s1,s2,s3,s4] (preserves selection order)
    - Given selections from 1 source, returns plan with empty secondarySources

    assembleMultiSourceDeck (single-source fast path):
    - Given selections all from one source, delegates to assembleDeckFromSlides and returns its result
    - Does NOT call any Drive/Slides API directly for single-source case
  </behavior>

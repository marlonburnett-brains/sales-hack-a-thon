Register the following API routes using the existing `registerApiRoute` pattern in `apps/agent/src/mastra/index.ts`:

1. **GET /agent-configs** -- List all agent configs with published version info and draft status. Query all AgentConfig records with `include: { publishedVersion: true, versions: { orderBy: { version: "desc" }, take: 1 } }`. For each config, compute `hasDraft` by checking if the latest version exists and `isPublished === false`. Return array of objects: `{ agentId, name, responsibility, family, isShared, publishedVersion: number|null, hasDraft: boolean, draftVersion: number|null }`.

2. **GET /agent-configs/:agentId** -- Get single agent config with published version details and latest draft (if any). Find by `agentId` with `include: { publishedVersion: true, versions: { orderBy: { version: "desc" }, take: 1 } }`. Return: `{ agentId, name, responsibility, family, isShared, publishedVersion: { id, version, baselinePrompt, rolePrompt, compiledPrompt, changeSummary, publishedAt, publishedBy } | null, draft: { id, version, rolePrompt, createdAt } | null }`.

3. **GET /agent-configs/:agentId/versions** -- List all versions for an agent ordered by version desc. Include: version number, changeSummary, isPublished, publishedAt, publishedBy, createdAt, rolePrompt (for diff).

4. **POST /agent-configs/:agentId/draft** -- Create a draft version. Accept `{ rolePrompt: string, userId?: string }`. Find the next version number (max existing version + 1). Get current published baseline. Compile with `compileAgentInstructions(baseline, rolePrompt)`. Create `AgentConfigVersion` with `isPublished: false`. Use optimistic locking: accept `expectedVersion` in body, reject if latest version !== expectedVersion. Return the new draft version.

5. **POST /agent-configs/:agentId/publish** -- Publish the latest draft. Accept `{ changeSummary?: string, userId?: string }`. Find latest unpublished version, set `isPublished: true, publishedAt: new Date(), publishedBy: userId, changeSummary`. Update `AgentConfig.publishedVersionId` to the new version. Call `invalidateAgentPromptCache({ agentId })`. Return updated config.

6. **POST /agent-configs/:agentId/discard** -- Discard the latest draft. Find latest version where `isPublished === false`. Delete it. Return `{ success: true }`.

7. **POST /agent-configs/:agentId/rollback** -- Accept `{ targetVersion: number, userId?: string }`. Find the target version. Create a new version with the target's `rolePrompt`, current published `baselinePrompt`, recompiled `compiledPrompt`. Set `changeSummary: "Rollback to v{targetVersion}"`, `isPublished: true, publishedAt: new Date(), publishedBy: userId`. Update `AgentConfig.publishedVersionId`. Call `invalidateAgentPromptCache({ agentId })`.

8. **POST /agent-configs/baseline/draft** -- Accept `{ baselinePrompt: string, userId?: string }`. Save the new baseline text as a draft. Use a virtual "shared-baseline" agent config (create one if it does not exist during seed or on first request). Create a new version with the baseline as both baseline and role prompt.

9. **POST /agent-configs/baseline/publish** -- Accept `{ changeSummary?: string, userId?: string }`. Get the latest baseline draft. For ALL agent configs that have a published version, create a new version with the new baseline + existing rolePrompt, compile, set isPublished: true. Update all `publishedVersionId` pointers. Call `invalidateAgentPromptCache()` (no args, clears entire cache). Return `{ agentsUpdated: number }`.

Import `compileAgentInstructions` and `invalidateAgentPromptCache` from `../lib/agent-config`. Import `prisma` from `../lib/db`.

Use `z.object(...)` schemas for request body validation on POST routes, following the existing pattern in the file.

  <action>
    Create `apps/agent/src/ingestion/run-ingestion.ts` -- the full bulk ingestion pipeline, runnable via `npx tsx apps/agent/src/ingestion/run-ingestion.ts`.

    The script orchestrates the complete pipeline:

    **Phase A -- Discovery and extraction:**
    1. Read the approved solution pillar list from `apps/agent/src/ingestion/manifest/solution-pillars.json` (generated by Plan 02-01 pilot)
    2. Call `discoverPresentations(env.GOOGLE_DRIVE_FOLDER_ID)` to find ALL presentations in the Drive folder
    3. Log total presentations found and their folder distribution
    4. Call `extractAllSlides(presentations)` to extract slide-level text from every presentation
    5. Log total slides extracted, count of low-content slides

    **Phase B -- Classification:**
    6. Call `classifyAllSlides(slides, solutionPillarList)` to assign metadata to every slide (metadata now includes `subsectors` field)
    7. Apply content_type overrides based on known presentation names from CONTEXT.md:
       - "Two Pager Template" -> content_type: template, touchType: [touch_1]
       - "Meet Lumenalta - 2026" -> content_type: template, touchType: [touch_2]
       - "NBCUniversal", "Bleecker Street Group" -> content_type: example, touchType: [touch_2]
       - "20251021 Master Solutions", "2026 GTM Solutions", "200A Master Deck" -> content_type: template, touchType: [touch_3]
       - "Alaska Airlines", "MasterControl", "Encompass", "WSA", "Satellite Industries", "Gravie" -> content_type: example, touchType: [touch_4]
       - All other decks: use Gemini classification as-is
    8. For case study decks (detected by folder name containing "case stud" or presentation name containing "case stud"): override content_type to "case_study"

    **Phase C -- Manifest generation:**
    9. Write `content-manifest.json` with every classified slide in the format from RESEARCH.md (documentId, presentationId, presentationName, slideObjectId, slideIndex, textPreview, speakerNotesPreview, isLowContent, metadata, status: "pending"). The metadata object MUST include the `subsectors` array.
    10. Generate `coverage-report.json`:
        ```json
        {
          "generatedAt": "ISO timestamp",
          "totalPresentations": N,
          "totalSlides": N,
          "lowContentSlides": N,
          "byIndustry": {
            "Consumer Products": { "templates": N, "examples": N, "caseStudies": N, "totalSlides": N },
            ...for all 11 industries
          },
          "byFunnelStage": { "First Contact": N, "Intro Conversation": N, ... },
          "byContentType": { "template": N, "example": N, "case_study": N, ... },
          "gaps": ["Industries with 0 templates or 0 case studies"]
        }
        ```
    11. Log the coverage report summary to console: industries covered, gaps found, total by content type

    **Phase D -- Bulk ingestion:**
    12. Read all "pending" entries from content-manifest.json
    13. For each slide, call `ingestDocument()` from atlusai-client.ts
    14. Use deterministic document IDs for idempotency -- check existence before creating (or upsert if AtlusAI supports it)
    15. Process in batches of 10 with a 500ms pause between batches (respect API rate limits)
    16. Update manifest status to "ingested" after each successful ingestion
    17. On error, log the error and mark status as "error" -- continue with remaining slides
    18. After all slides processed, write the updated manifest back to content-manifest.json
    19. Log final summary: total ingested, total skipped (already existed), total errors

    **Phase E -- Verification queries:**
    20. Run 3 verification queries against AtlusAI after ingestion:
        - Semantic search: "healthcare digital transformation" -- expect at least 1 result
        - Semantic search: "financial services case study" -- expect at least 1 result
        - Structured filter (if AtlusAI supports it): industry = "Consumer Products" -- expect at least 1 result
    21. Log verification results

    Support a `--manifest-only` CLI flag that runs Phases A-C only (no ingestion) for pre-ingestion review. Support a `--ingest-only` CLI flag that runs Phase D only (reads existing manifest).
  </action>

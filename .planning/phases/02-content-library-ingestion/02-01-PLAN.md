    **Step 1 — AtlusAI MCP tool discovery (BLOCKING — must resolve first):**
    Create `apps/agent/src/lib/atlusai-client.ts` using `@mastra/mcp` MCPClient. Install `@mastra/mcp@^1.0.2` in apps/agent first.

    Connect to the AtlusAI SSE endpoint at `https://knowledge-base-api.lumenalta.com/sse` and call `listTools()` to enumerate ALL available MCP tools — not just the 3 whitelisted in `.claude/settings.local.json`.

    Write a standalone discovery script that can be run via `npx tsx apps/agent/src/lib/atlusai-client.ts` and logs the full tool inventory to console. Document every tool name, description, and input schema.

    Based on what tools are found:
    - If a document creation tool exists (e.g., `knowledge_base_create_document`, `knowledge_base_ingest`, `upsert_document`): wrap it in an `ingestDocument()` export function.
    - If NO creation tools exist: investigate whether AtlusAI re-indexes content from the connected Google Drive folder (the ~9,642 existing docs were ingested from Drive). The fallback is creating structured Google Docs in the connected Drive folder with metadata in document properties. Document findings in a comment block at the top of the file.

    Export: `createAtlusAIClient()`, `discoverAtlusAITools()`, `ingestDocument(doc: SlideDocument): Promise<void>`. The `ingestDocument` function signature will depend on what tools are discovered.

    **Step 2 — Drive content discovery:**
    Create `apps/agent/src/ingestion/discover-content.ts`. Use `getDriveClient()` from `google-auth.ts`.

    Implement `discoverPresentations(folderId: string)` that:
    - Lists all Google Slides presentations (mimeType = 'application/vnd.google-apps.presentation') in the given folder
    - Recursively traverses subfolders (mimeType = 'application/vnd.google-apps.folder')
    - Handles pagination correctly (follow `nextPageToken` until exhausted)
    - Uses `supportsAllDrives: true` and `includeItemsFromAllDrives: true` on all Drive API calls (locked pattern from Phase 1)
    - Returns `DrivePresentation[]` with `{ id, name, mimeType, folderPath }` where `folderPath` is the slash-separated path of parent folder names (useful for classification context)
    - Adds a 200ms delay between API calls to avoid rate limits

    Export: `discoverPresentations`, `DrivePresentation` type.

    **Step 3 — Slide text extraction library:**
    Create `apps/agent/src/lib/slide-extractor.ts`. Use `getSlidesClient()` from `google-auth.ts`.

    Implement `extractSlidesFromPresentation(presentationId: string, presentationName: string, folderPath: string)` that:
    - Calls `slides.presentations.get({ presentationId })` to get ALL slides in one API call
    - For each slide, extracts text content by iterating pageElements and concatenating TextRun content strings
    - For each slide, extracts speaker notes from `slide.slideProperties.notesPage` — check for `speakerNotesObjectId` existence before accessing (may be undefined if no notes exist)
    - Returns `ExtractedSlide[]` with shape:
      ```typescript
      interface ExtractedSlide {
        documentId: string          // SHA-256 of presentationId:slideObjectId (first 32 chars)
        presentationId: string
        presentationName: string
        folderPath: string
        slideObjectId: string
        slideIndex: number
        textContent: string
        speakerNotes: string
        isLowContent: boolean       // true if combined text + notes < 20 chars
      }
      ```
    - Uses `createHash('sha256')` from `node:crypto` for deterministic document IDs

    Export: `extractSlidesFromPresentation`, `ExtractedSlide` type.

    **Step 4 — Batch extraction orchestrator:**
    Create `apps/agent/src/ingestion/extract-slides.ts` that imports `extractSlidesFromPresentation` and processes an array of `DrivePresentation[]`.
    - Process presentations sequentially (one at a time) with a 200ms delay between API calls to respect rate limits
    - Log progress: "Extracting {name} ({index}/{total})..."
    - Return the flat array of all `ExtractedSlide[]` across all presentations

    Export: `extractAllSlides`.

    **Step 5 — Add GEMINI_API_KEY to env.ts:**
    Add `GEMINI_API_KEY: z.string().min(1)` to the server section of `apps/agent/src/env.ts`. This is needed for Gemini metadata classification in the pilot and bulk ingestion.

    **Step 2 — Metadata classification with Gemini:**
    Create `apps/agent/src/ingestion/classify-metadata.ts`.

    Define `SlideMetadataSchema` as a Zod object with these fields (all Gemini-safe: flat, no transforms, no unions):
    - `industries`: array of enum — the 11 industries from CONTEXT.md: Consumer Products, Education, Financial Services & Insurance, Health Care, Industrial Goods, Private Equity, Public Sector, Technology Media & Telecommunications, Transportation & Logistics, Travel & Tourism, Professional Services
    - `solutionPillars`: array of strings — use exact names from the pillar list (derived from Master Solutions deck)
    - `funnelStages`: array of enum — First Contact, Intro Conversation, Capability Alignment, Solution Proposal (multi-value per locked decision)
    - `contentType`: enum — template, example, case_study, brand_guide, resource
    - `slideCategory`: enum — title, divider, industry_overview, capability_description, case_study_problem, case_study_solution, case_study_outcome, team_intro, methodology, timeline, pricing, next_steps, appendix, other
    - `buyerPersonas`: array of enum — CIO, CTO, CFO, VP Engineering, VP Data, VP Product, VP Operations, CEO, General
    - `touchType`: array of enum — touch_1, touch_2, touch_3, touch_4

    Implement `classifySlide(slide: ExtractedSlide, titleSlideText: string, solutionPillarList: string[])` that:
    - Uses `@google/genai` GoogleGenAI client with model `gemini-2.5-flash` (stable, not preview — per research recommendation)
    - Passes deck-level context in the prompt: presentation name, folder path, title slide text (first slide content)
    - Uses `responseMimeType: "application/json"` and `responseSchema` from zod-to-json-schema for structured output
    - Returns the classified metadata merged with the ExtractedSlide data

    Implement `classifyAllSlides(slides: ExtractedSlide[], solutionPillarList: string[])` that:
    - Groups slides by presentation (all slides from same deck share title slide context)
    - Classifies slides sequentially within each presentation (Gemini rate limits)
    - Adds 300ms delay between Gemini calls
    - For low-content slides (isLowContent=true), still classify but the prompt should note "this is a title/divider slide with minimal text"

    Export: `SlideMetadataSchema`, `ClassifiedSlide`, `classifySlide`, `classifyAllSlides`.

    **Step 3 — Pilot ingestion script:**
    Create `apps/agent/src/ingestion/pilot-ingestion.ts` — a runnable script (`npx tsx apps/agent/src/ingestion/pilot-ingestion.ts`).

    The pilot script:
    1. Discovers all presentations in the Drive folder
    2. Extracts the solution pillar taxonomy from the Master Solutions deck and GTM Solutions deck by:
       - Finding these two decks by name in the discovered presentations
       - Extracting all slides from both
       - Using Gemini to identify the list of unique solution pillar names mentioned across all slides
       - Writing the pillar list to `apps/agent/src/ingestion/manifest/solution-pillars.json`
    3. Selects 2-3 pilot decks: the "Meet Lumenalta - 2026" deck (Touch 2 template), one example proposal, and one case study (if discoverable by name)
    4. Extracts all slides from the pilot decks
    5. Classifies metadata for all pilot slides using Gemini with the extracted pillar list
    6. Writes a pilot manifest to `apps/agent/src/ingestion/manifest/pilot-manifest.json` in this format per slide:

    7. Ingests the pilot slides into AtlusAI using the `ingestDocument()` function from atlusai-client.ts
    8. Verifies ingestion by calling `knowledge_base_search_semantic` for a query related to the pilot deck content
    9. Logs a summary: total slides extracted, total classified, total ingested, any errors

    Add the `manifest/` directory to .gitignore (it contains generated content that should not be committed).

Update `apps/web/src/app/(authenticated)/layout.tsx` to use the new Sidebar component instead of the inline top nav bar.

The layout remains a Server Component for the auth check. It passes user data as props to the Sidebar client component:

```typescript
import { redirect } from "next/navigation";
import { createClient } from "@/lib/supabase/server";
import { Sidebar } from "@/components/sidebar";

export default async function AuthenticatedLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  const supabase = await createClient();
  const { data: { user } } = await supabase.auth.getUser();

  if (!user) {
    redirect("/login");
  }

  const name = (user.user_metadata?.full_name as string) ?? "User";
  const email = user.email ?? "";
  const avatarUrl = (user.user_metadata?.avatar_url as string) ?? "";

  return (
    <Sidebar user={{ name, email, avatarUrl }}>
      {children}
    </Sidebar>
  );
}
```

**Key changes:**
- Remove the entire `<nav>` top bar and `<main>` wrapper
- Remove Briefcase and UserNav imports (Sidebar handles these internally)
- Sidebar component wraps `{children}` and provides both navigation and content area
- The main content padding/max-width is handled inside Sidebar's main area

**Verification:** After this change, navigate to `/deals` -- the existing deals page content should render correctly inside the new sidebar layout with no visual breakage.

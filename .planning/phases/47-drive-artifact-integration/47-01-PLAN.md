model UserSetting {
  id        String   @id @default(cuid())
  userId    String
  key       String   // "drive_root_folder_id", "drive_root_folder_name"
  value     String
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  @@unique([userId, key])
  @@index([userId])
}

**drive-folders.ts** -- Add 4 new functions, keep existing exports:

1. `resolveRootFolderId(userId?: string): Promise<string>` -- Query UserSetting for key="drive_root_folder_id" by userId. If found, return value. Otherwise fall back to `env.GOOGLE_DRIVE_FOLDER_ID` (which may be empty string now). Import prisma client at top.

2. `shareWithOrg(params: { fileId: string; ownerEmail?: string }): Promise<void>` -- Replaces makePubliclyViewable pattern. Three permission calls:
   - `type: "domain", domain: "lumenalta.com", role: "reader"` (org-wide viewer)
   - `type: "user", emailAddress: ownerEmail, role: "writer"` (deal owner editor, only if ownerEmail provided)
   - Service account already has access as file creator, no extra permission needed.
   - All calls must include `supportsAllDrives: true` and `sendNotificationEmail: false` per research pitfall #5.

3. `getOrCreateSubfolder(parentFolderId: string, folderName: string): Promise<string>` -- Same idempotent check-before-create pattern as getOrCreateDealFolder but for any subfolder name (used for "Archive" subfolder).

4. `archiveExistingFile(params: { dealFolderId: string; fileId: string }): Promise<void>` -- Get or create "Archive" subfolder inside deal folder, then move file from deal folder into Archive using `drive.files.update` with `addParents`/`removeParents`.

Keep `makePubliclyViewable` exported (do NOT remove yet -- Plan 03 will replace all call sites then remove it).

**env.ts** -- Change `GOOGLE_DRIVE_FOLDER_ID` from `z.string().min(1)` to `z.string().default('')` so it is optional. This prevents agent startup failures when users rely on UserSetting instead.

**OAuth scope upgrade** -- In all 3 files, replace `drive.readonly` with `drive` (full scope):
- `apps/web/src/app/login/page.tsx` line 54: change `auth/drive.readonly` to `auth/drive`
- `apps/web/src/app/(authenticated)/actions/actions-client.tsx` line 166: same change
- `apps/web/src/components/user-nav.tsx` line 60: same change

This is required because folder creation and permission management need full Drive access, and the Picker API needs at least `drive.file` scope.

  <action>
Replace the placeholder DiscoveryClient with the full implementation. Start with browse mode:

**Component structure:**
```
DiscoveryClient
  - Header: "AtlusAI Discovery" title
  - Search bar (full width, with Search icon from lucide-react)
  - View toggle buttons (Grid/List icons from lucide-react: LayoutGrid, List)
  - Results area:
    - Browse mode: card grid OR list view (based on toggle)
    - Search mode: search results (Task 2)
  - Sentinel div for infinite scroll
  - Preview side panel (Task 2)
```

**Props:** `{ initialBrowse: BrowseResult }`

**State:**
- `documents: DiscoveryDocument[]` -- initialized from `initialBrowse.documents`
- `ingestedHashes: Set<string>` -- initialized from `initialBrowse.ingestedHashes`
- `nextCursor: string | undefined` -- from `initialBrowse.nextCursor`
- `hasMore: boolean` -- true if nextCursor exists
- `isLoadingMore: boolean` -- guard for infinite scroll
- `viewMode: "grid" | "list"` -- default "grid"
- `searchQuery: string` -- controlled input
- `mode: "browse" | "search"` -- derived from searchQuery
- `searchResults: DiscoveryDocument[]` -- populated by search
- `isSearching: boolean` -- loading state for search
- `selectedPreview: DiscoveryDocument | null` -- for side panel

**Search bar:**
- Full-width input with `Search` icon on the left, `X` clear button on right when non-empty
- Placeholder: "Search AtlusAI content..."
- On change: set `searchQuery`. If empty, set `mode` to "browse". If non-empty, debounce 300ms then call `searchDocumentsAction(query)`, set `mode` to "search".
- Use `useRef<NodeJS.Timeout>(null)` for debounce timer. Clear previous timer on each keystroke.
- Pass the current value directly into the setTimeout callback to avoid stale closure (per pitfall #3 in RESEARCH.md).

**View toggle:**
- Two icon buttons (LayoutGrid / List from lucide-react) with active state highlighting
- Positioned to the right of the search bar on the same row
- Only visible in browse mode (search always shows list-style results)

**Card grid view (browse mode, viewMode="grid"):**
- Responsive grid: `grid-cols-1 sm:grid-cols-2 md:grid-cols-3 lg:grid-cols-4 gap-4`
- Each card:
  - White bg, rounded-lg border, shadow-sm, hover:shadow-md transition
  - Document title (truncated to 2 lines, `line-clamp-2 font-medium text-sm text-slate-900`)
  - Content preview (truncated to 3 lines, `line-clamp-3 text-xs text-slate-500 mt-1`)
  - Bottom row: source badge if available, "Ingested" green badge if contentHash in ingestedHashes
  - Checkbox in top-right corner (for Plan 03 selection -- render as disabled placeholder for now with `opacity-30`)
  - `onClick` -> set `selectedPreview` to open side panel
  - `cursor-pointer` on the card

**List view (browse mode, viewMode="list"):**
- Table-like layout using div rows (not actual table element)
- Header row: Title | Preview | Status
- Each row: title (truncated), content preview (truncated, `max-w-md`), ingested status badge
- Alternating row background (`even:bg-slate-50`)
- `cursor-pointer`, `hover:bg-slate-100` on each row
- `onClick` -> set `selectedPreview`

**Infinite scroll:**
- Sentinel `<div ref={sentinelRef} className="h-1" />` at bottom of results
- IntersectionObserver with `rootMargin: "200px"`:
  - Only observe when `mode === "browse"` and `hasMore` and `!isLoadingMore`
  - On intersection: set `isLoadingMore = true`, call `browseDocumentsAction({ cursor: nextCursor, limit: 20 })`, append documents, update cursor/hasMore, set `isLoadingMore = false`
- Show 4 skeleton cards when loading more (simple pulsing `bg-slate-100 rounded-lg h-40 animate-pulse`)
- Clean up observer on unmount or mode change

**Empty states:**
- Browse with no documents: "No documents found in AtlusAI" with Inbox icon
- Search with no results: "No results for '[query]'. Try broader terms like 'healthcare solutions', 'case study', or 'capabilities overview'."

**Already-ingested badge:**
- Compute `isIngested` by generating SHA-256 hash of `doc.textContent` and checking against `ingestedHashes` set
- Use the Web Crypto API: `crypto.subtle.digest('SHA-256', new TextEncoder().encode(text))` then convert to hex
- Show green badge: `<span className="inline-flex items-center rounded-full bg-green-50 px-2 py-0.5 text-xs font-medium text-green-700">Ingested</span>`
- Hashing can be expensive -- memoize with `useMemo` keyed on document list + ingestedHashes

NOTE: For the content hash comparison to work, the agent must compute the same hash. Check how `contentHash` is set in `ingest-template.ts` and use the same algorithm on the client. If the hash is computed differently (e.g., server uses Node crypto), compute on the server side instead and return `isIngested: boolean` per document from the agent endpoint. This is the PREFERRED approach: have the agent's browse/search endpoints return an `ingestedDocIds: string[]` (matching by slideId or documentTitle) rather than raw hashes.

Extend the DiscoveryClient from Plan 02 to add selection, ingestion, and progress tracking:

**New state:**
- `selectedIds: Set<string>` -- set of slideIds currently selected
- `itemStatuses: Map<string, "idle" | "pending" | "ingesting" | "done" | "error">` -- per-item ingestion status
- `itemErrors: Map<string, string>` -- error messages for failed items
- `activeBatchId: string | null` -- current batch being polled
- `isIngesting: boolean` -- true while any ingestion is in progress

**Enable checkboxes on cards and list items:**
- Replace the disabled placeholder checkboxes from Plan 02 with functional ones
- Each card/list item gets a checkbox in the top-right (grid) or left side (list)
- Checkbox is DISABLED + CHECKED with green "Ingested" badge if item is already ingested (from `ingestedHashes` or per-document `isIngested` flag)
- Checkbox is DISABLED if item status is "pending" or "ingesting" (currently being ingested)
- Checkbox is DISABLED + shows green check if status is "done"
- Otherwise: functional checkbox that toggles `selectedIds`
- Checkbox click must `stopPropagation()` to prevent opening the preview panel
- Use `<input type="checkbox" className="h-4 w-4 rounded border-slate-300 text-blue-600 focus:ring-blue-500 cursor-pointer" />`

**Status indicators on items:**
- `"pending"`: yellow dot + "Queued" text (`text-yellow-600`)
- `"ingesting"`: blue spinner animation + "Ingesting..." (`text-blue-600 animate-spin` on a Loader2 icon from lucide-react)
- `"done"`: green check + "Ingested" badge (same as already-ingested)
- `"error"`: red X + "Failed" badge with tooltip showing error message. Use `title` attribute on the badge for tooltip. Add a small "Retry" button (RotateCcw icon from lucide-react) next to the error badge.

**Floating toolbar:**
- Visible when `selectedIds.size > 0`
- Positioned: `fixed bottom-6 left-1/2 z-40 -translate-x-1/2`
- Styling: `rounded-lg border border-slate-200 bg-white px-6 py-3 shadow-lg flex items-center gap-4`
- Content:
  - Selection count: `"{n} selected"` (`text-sm font-medium text-slate-700`)
  - Clear button: "Clear" text button (`text-sm text-slate-500 hover:text-slate-700`)
  - Ingest button: `"Ingest {n} selected"` blue button (`bg-blue-600 text-white rounded-md px-4 py-2 text-sm font-medium hover:bg-blue-700`)
  - During ingestion: show progress text instead of ingest button: `"Ingesting... {done}/{total}"` with disabled state
- Slide-up animation: `transition-transform duration-200` -- translate-y from bottom when appearing
- If `isIngesting`, the Ingest button shows a Loader2 spinner and is disabled

**Batch ingestion flow (`handleBatchIngest`):**
1. Collect selected DiscoveryDocument objects from the current documents/searchResults arrays
2. Set all selected items to "pending" in `itemStatuses`
3. Set `isIngesting = true`
4. Call `startDiscoveryIngestionAction(selectedItems)` to get `batchId`
5. Set `activeBatchId = batchId`
6. Clear `selectedIds` (items are now tracked by status, not selection)
7. Start polling interval (2 seconds) using `setInterval`. On each tick call `getDiscoveryIngestionProgressAction(batchId)`, update `itemStatuses` and `itemErrors` from the response. When `progress.complete` is true, clear the interval, set `isIngesting = false`, show toast notification.
8. Store interval ref to clean up on unmount

**Individual retry for failed items:**
- "Retry" button next to error badge
- On click: call `startDiscoveryIngestionAction([failedItem])` with just that one item
- Start a new polling interval for the single-item batch
- Update status to "pending" immediately

**Preview panel Ingest button:**
- Replace the disabled placeholder "Ingest" button from Plan 02 with a functional one
- If preview item is already ingested or has "done" status: show "Already Ingested" disabled button
- If preview item is pending/ingesting: show "Ingesting..." with spinner
- Otherwise: show "Ingest" button that adds the item to selection and triggers `handleBatchIngest` for just that item

**Toast notifications (sonner):**
- Import `toast` from `sonner`
- On ingestion complete: `toast.success("Ingestion complete: N items ingested")`
- On ingestion error (all items failed): `toast.error("Ingestion failed. Check individual items for details.")`
- On retry start: `toast.info("Retrying ingestion...")`

**Cleanup:**
- `useEffect` cleanup: clear polling interval on unmount
- `useRef` for interval ID to avoid stale closures

**Error resilience (per CONTEXT.md):**
- Failed items show error badge -- other items continue
- No batch abort mechanism
- User can retry failed items individually

  <action>
Create a deploy/ directory at the monorepo root with the following files:

deploy/Dockerfile -- Multi-stage build for the Mastra agent server:
- Builder stage: FROM node:22-alpine AS builder
  - WORKDIR /app
  - Copy monorepo root files: package.json, pnpm-lock.yaml, pnpm-workspace.yaml
  - Copy packages/ directory (workspace dependencies including @lumenalta/schemas)
  - Copy apps/agent/ directory
  - Enable corepack: RUN corepack enable and pnpm install --frozen-lockfile
  - Change to agent dir: WORKDIR /app/apps/agent
  - Generate Prisma client: RUN npx prisma generate
  - Build agent: RUN pnpm build
- Production stage: FROM node:22-alpine
  - Install gcompat for native module support: RUN apk add --no-cache gcompat
  - WORKDIR /app
  - Copy mastra build output: COPY --from=builder /app/apps/agent/.mastra/output/ ./
  - Copy Prisma client (runtime native addon): COPY --from=builder /app/apps/agent/node_modules/.prisma/ ./node_modules/.prisma/ and COPY --from=builder /app/apps/agent/node_modules/@prisma/ ./node_modules/@prisma/
  - Copy entrypoint: COPY deploy/entrypoint.sh /entrypoint.sh and RUN chmod +x /entrypoint.sh
  - Create non-root user: RUN addgroup -g 1001 -S nodejs and adduser -S mastra -u 1001 and chown -R mastra:nodejs /app
  - USER mastra
  - ENV NODE_ENV=production and ENV PORT=4111
  - EXPOSE 4111
  - ENTRYPOINT ["/entrypoint.sh"]
  - CMD ["node", "index.mjs"]

deploy/entrypoint.sh -- Writes Vertex AI credentials to a temp file:
- Shebang: #!/bin/sh
- If VERTEX_SERVICE_ACCOUNT_KEY is set, write it to /tmp/vertex-credentials.json and export GOOGLE_APPLICATION_CREDENTIALS=/tmp/vertex-credentials.json
- exec "$@" to pass through to CMD

deploy/docker-compose.yml -- Two services (app + caddy):
- app service: builds from deploy/Dockerfile with context . (monorepo root), uses env_file: .env, restart: unless-stopped, exposes port 4111 (internal only, not published to host), includes healthcheck using wget --spider -q http://localhost:4111/health with 30s interval, 5s timeout, 3 retries, 10s start_period
- caddy service: image caddy:2-alpine, ports 80:80 and 443:443 and 443:443/udp (for HTTP/3), volume mounts for ./deploy/Caddyfile:/etc/caddy/Caddyfile, caddy_data:/data, caddy_config:/config, restart: unless-stopped, depends_on app
- Named volumes: caddy_data, caddy_config

IMPORTANT: The docker-compose.yml build.context must be . (monorepo root) because the Dockerfile needs access to packages/, apps/agent/, pnpm-lock.yaml, etc. The dockerfile path is deploy/Dockerfile.

deploy/Caddyfile -- Reverse proxy config:
- Use {$AGENT_DOMAIN:localhost} as the site address so the domain is configurable via environment variable
- Single directive: reverse_proxy app:4111

deploy/deploy.sh -- SSH deployment script:
- Shebang: #!/bin/bash with set -euo pipefail
- Accept SSH_HOST as required first argument, SSH_USER as optional second (default: ubuntu)
- SSH into the VM and run: cd /opt/lumenalta, git pull origin main, docker compose build --no-cache, docker compose down, docker compose up -d
- Wait 8 seconds then curl http://localhost:4111/health to verify
- Print success or failure message
- Make executable with chmod +x

Include every env var from apps/agent/src/env.ts PLUS VERTEX_SERVICE_ACCOUNT_KEY (new, for Docker entrypoint to write Vertex AI credentials file) PLUS AGENT_DOMAIN (for Caddyfile domain).

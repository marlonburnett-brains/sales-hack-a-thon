<objective>
Implement the Section Matcher -- a deterministic scoring algorithm that selects the best candidate slide for each section in a GenerationBlueprint based on deal context metadata, with vector similarity tiebreaker and cross-touch exclusion.

Purpose: This is the bridge between the Blueprint Resolver (Phase 51) and the Multi-Source Assembler (Phase 52). Without it, the pipeline has candidate slides but no way to pick which one goes in each section.

Output: `selectSlidesForBlueprint()` function that takes BlueprintWithCandidates and returns SectionMatchResult (SlideSelectionPlan + mutated blueprint with selections filled).
</objective>

```typescript
export interface SectionMatchResult {
  plan: SlideSelectionPlan;
  blueprint: GenerationBlueprint;
}

export async function selectSlidesForBlueprint(
  input: BlueprintWithCandidates,
): Promise<SectionMatchResult>
```

Implementation details:
1. **Cross-touch exclusion (FR-3.6):** Before scoring, filter each section's candidateSlideIds by removing IDs in `input.blueprint.dealContext.priorTouchSlideIds`. If filtering removes ALL candidates, fall back to original unfiltered list.

2. **Metadata scoring (FR-3.1):** For each candidate, parse `classificationJson` as SlideMetadata (try/catch, score 0 on failure). Score using weights: `{ industry: 3, pillar: 3, persona: 2, funnelStage: 2 }`. Cap pillar overlap at `Math.min(overlapCount, 2) * weight`. Use `parsed.industries?.includes()` for industry, `parsed.buyerPersonas?.includes()` for persona, `parsed.funnelStages?.includes()` for funnel stage.

3. **Sparse fallback (FR-3.4):** If ALL candidates in a section score 0 on metadata, sort by `candidate.confidence ?? 0` descending. If still all zero, pick first candidate.

4. **Vector tiebreaker (FR-3.2):** If top 2+ candidates have equal metadata score AND metadata score > 0, generate a deal context embedding using `generateEmbedding(buildDealContextText(ctx))` (lazy -- only on first tie, cached for subsequent ties). Query `prisma.$queryRaw` with pgvector `<=>` operator for tied candidate IDs. Pick highest similarity. Import `prisma` from `../lib/db` and `generateEmbedding` from `../ingestion/embed-slide`.

5. **Build matchRationale string:** E.g., "Industry match (Healthcare) + 2 pillar overlap + persona match. Score: 10" or "Fallback: highest confidence (0.92)" or "Tiebreaker: vector similarity 0.87 vs 0.82".

6. **Output:** Build `SlideSelectionPlan.selections[]` with one entry per filled section. Mutate `blueprint.sections[i].selectedSlideId` and `sourcePresentationId`. Return `{ plan, blueprint }`.

Helper function for deal context text:
```typescript
function buildDealContextText(ctx: DealContext): string {
  return [ctx.industry, ...ctx.pillars, ctx.persona, ctx.funnelStage, ctx.companyName]
    .filter(Boolean).join(" | ");
}
```

Anti-patterns to avoid:
- Do NOT use LLM calls for selection (deterministic scoring only)
- Do NOT mutate the candidates Map from Phase 51
- Do NOT do global vector searches -- only within per-section candidate sets
- Do NOT query SlideEmbedding/Template per-candidate (use pre-resolved data from candidates Map)

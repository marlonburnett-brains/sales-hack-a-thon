  <action>
Create `apps/tutorials/scripts/upload-thumbnails.ts` as a one-off helper that honors the locked decision to use 1-second video frames and gcloud CLI upload:

- Read rendered MP4s from `apps/tutorials/output/videos`.
- Support `--dry-run` for verification and `--bucket <name>` override, defaulting to `GCS_TUTORIAL_BUCKET` or `atlusdeck-tutorials`.
- For each video, extract a single JPG frame at 1 second using `ffmpeg` into a temp/output thumbnails folder.
- Upload frames with `gcloud storage cp` to `gs://{bucket}/thumbnails/{slug}.jpg` (use gcloud CLI per locked decision rather than the Google APIs upload path here).
- Write `apps/tutorials/output/tutorial-thumbnails-manifest.json` with `{ slug, thumbnailUrl }[]`, using public URLs in the form `https://storage.googleapis.com/{bucket}/thumbnails/{slug}.jpg`.
- Continue past individual failures, print a success/failure summary, and exit non-zero only when a non-dry-run upload run has failures.

Update `apps/agent/prisma/seed.ts` to read `tutorial-thumbnails-manifest.json` if present and merge `thumbnailUrl` into each `prisma.tutorial.upsert(...)`. If the manifest is missing, log a warning and keep `thumbnailUrl: null` rather than failing the seed. Do not touch video `gcsUrl`, watched state, or any Phase 73 playback logic.
  </action>

Purpose: This is the user-facing feature that brings deck intelligence to life. Users see AI-analyzed deck patterns and can refine them through conversation.
Output: Complete deck structure UI with chat refinement, streaming proxy, agent chat endpoint.

  <action>
    1. **Create chat-refinement.ts** on agent side:
       - `streamChatRefinement(touchType, userMessage, chatHistory, onChunk)` async function:
         a. Load existing DeckStructure for the touchType.
         b. Build chat prompt: include current structureJson as context, chatContextJson as past constraints, last 5 messages from chatHistory, and the new userMessage. Instruct LLM to: analyze the user's feedback, explain what changes it would make, and output a structured response.
         c. Use Google GenAI `generateContentStream` for streaming response. For each chunk, call onChunk(text). Collect full response text.
         d. After streaming completes, re-run inference with updated constraints: take the full AI response, summarize key refinements into a constraint string, append to existing chatContextJson, then call `inferDeckStructure(touchType, updatedConstraints)`.
         e. Save the user message and assistant response as DeckChatMessage records. For the assistant message, compute structureDiff by comparing old sections vs new sections (added/modified/removed section names).
         f. Update `lastChatAt` on DeckStructure to now (active session protection).
         g. Return { aiResponse: string, updatedStructure: DeckStructureOutput, diff: { added: string[], modified: string[], removed: string[] } }.
       - **Chat context summarization:** If total chatMessages count exceeds 10, summarize the oldest messages into a constraints string and store in chatContextJson. Only keep last 6 messages unsummarized. This prevents context window explosion per CONTEXT.md decision.
       - Handle Google GenAI streaming: if `generateContentStream` fails with Vertex AI, fall back to non-streaming `generateContent` and simulate streaming by yielding the full response in one chunk (per RESEARCH.md open question 1).

    2. **Register chat endpoint** in mastra/index.ts:
       - `POST /deck-structures/:touchType/chat` -- accepts { message: string }. Uses Hono streaming (`stream` from "hono/streaming") to:
         a. Call streamChatRefinement with onChunk that writes to the stream.
         b. After streaming text, write a final JSON delimiter `\n---STRUCTURE_UPDATE---\n` followed by JSON.stringify of { updatedStructure, diff }.
         c. This allows the client to separate streamed text from the structure update payload.
       - If `stream` from hono/streaming is not available in registerApiRoute context, fall back to a manual Response with ReadableStream (create ReadableStream with a controller, push chunks from onChunk, close when done).

    3. **Create Next.js streaming proxy** at `apps/web/src/app/api/deck-structures/chat/route.ts`:
       - Import env config (AGENT_SERVICE_URL, AGENT_API_KEY).
       - POST handler: read request body { touchType, message }, forward to agent `POST /deck-structures/${touchType}/chat` with Authorization Bearer header. Pipe agentRes.body directly back as Response with Content-Type: text/plain; charset=utf-8.
       - This avoids CORS issues per RESEARCH.md pitfall 1.

    4. **Add types to api-client.ts:**
       - `DeckStructureData` interface: { id, touchType, structureJson: string, exampleCount, confidence, chatContextJson, dataHash, inferredAt, lastChatAt, createdAt, updatedAt }.
       - `DeckChatMessageData` interface: { id, deckStructureId, role, content, structureDiff, createdAt }.
       - `getDeckStructures()` function: fetches GET /deck-structures from agent.
       - `getDeckStructure(touchType)` function: fetches GET /deck-structures/:touchType from agent, returns structure + chatMessages.

    5. **Create deck-structure-actions.ts** server actions:
       - "use server" directive.
       - `getDeckStructuresAction()`: calls getDeckStructures(), returns data.
       - `getDeckStructureAction(touchType)`: calls getDeckStructure(touchType), returns data.
       - `triggerInferenceAction(touchType)`: calls POST /deck-structures/:touchType/infer, returns result.

Create `apps/agent/src/generation/modification-executor.ts` implementing the `executeModifications` function. Follow the complete pattern from 55-RESEARCH.md "Complete Modification Executor Pattern" section with these specifics:

1. **Types**: Define and export `ExecuteModificationsParams` (presentationId: string, plans: ModificationPlan[]), `SlideModificationResult` (slideId, slideObjectId, status: "success"|"skipped"|"no_modifications", modificationsApplied, error?), and `ExecuteModificationsResult` (results[], totalApplied, totalSkipped).

2. **Import** `ModificationPlan` from `./modification-plan-schema` and `getSlidesClient` from `../lib/google-auth`. Import `slides_v1` type from `googleapis`.

3. **Main function** `executeModifications(params)`:
   - Get slides client via `getSlidesClient()`
   - Iterate `plans` sequentially (NOT Promise.all -- NFR-8)
   - For each plan:
     a. If `modifications.length === 0`, push "no_modifications" result, continue
     b. Wrap in try/catch (FR-6.4)
     c. Call `presentations.get({ presentationId })` to read current state
     d. Find target slide by `plan.slideObjectId` in response
     e. If slide not found, push "skipped" result, continue
     f. Build Set of current element IDs from `targetSlide.pageElements`
     g. For each modification, check `currentElementIds.has(mod.elementId)` -- skip if not found with console.warn
     h. For valid modifications, push `deleteText { objectId: mod.elementId, textRange: { type: "ALL" } }` and `insertText { objectId: mod.elementId, insertionIndex: 0, text: mod.newContent.replace(/\n$/, "") }`
     i. If requests not empty, call `presentations.batchUpdate({ presentationId, requestBody: { requests } })`
     j. Push "success" result with `modificationsApplied` count
   - In catch: `console.warn` the error, push "skipped" result
   - After loop: compute `totalApplied` (sum of modificationsApplied) and `totalSkipped` (count of "skipped" status)
   - Return `ExecuteModificationsResult`

4. **CRITICAL constraints**:
   - NEVER use `replaceAllText` anywhere in this module (NFR-7)
   - ALWAYS process slides sequentially with re-read between each (NFR-8)
   - ALWAYS trim trailing `\n` from newContent (Pitfall 5 from research)

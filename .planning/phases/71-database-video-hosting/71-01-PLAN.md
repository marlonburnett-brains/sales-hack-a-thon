<task type="auto">
  <name>Task 1: Add Tutorial, TutorialView, and AppFeedback models to Prisma schema</name>
  <files>apps/agent/prisma/schema.prisma</files>
  <action>
Append three new models to the end of `apps/agent/prisma/schema.prisma` (after the existing UserSetting model). Follow established conventions (cuid IDs, DateTime patterns, @@index on FKs).

**Tutorial model:**
- `id` String @id @default(cuid())
- `slug` String @unique — e.g., "getting-started", "deal-chat"
- `title` String — from fixture script.json
- `description` String — from fixture script.json
- `category` String — one of: "getting_started", "deal_workflows", "touch_points", "content_management", "review", "settings_admin"
- `gcsUrl` String — full public GCS URL
- `durationSec` Int — seconds, from ffprobe
- `sortOrder` Int — global 1-17
- `stepCount` Int @default(0) — number of steps from script.json
- `createdAt` DateTime @default(now())
- `updatedAt` DateTime @updatedAt
- `views` TutorialView[] — relation
- @@index([category])
- @@index([sortOrder])

**TutorialView model:**
- `id` String @id @default(cuid())
- `tutorialId` String — FK to Tutorial
- `tutorial` Tutorial @relation(fields: [tutorialId], references: [id])
- `userId` String — Supabase auth user ID
- `watched` Boolean @default(false) — true when video "ended" event fires
- `lastPosition` Float @default(0) — seconds into video for resume
- `watchedAt` DateTime? — when marked as watched
- `createdAt` DateTime @default(now())
- `updatedAt` DateTime @updatedAt
- @@unique([tutorialId, userId])
- @@index([userId])

**AppFeedback model:**
- `id` String @id @default(cuid())
- `sourceType` String — "tutorial" or future extensible values
- `sourceId` String — Tutorial ID or other entity ID
- `feedbackType` String — "tutorial_feedback" or "feature_feedback"
- `comment` String — free-text
- `userId` String? — nullable for anonymous
- `createdAt` DateTime @default(now())
- @@index([sourceType, sourceId])
- @@index([userId])

Add a section comment before the models: `// Phase 71: Tutorial system & feedback`

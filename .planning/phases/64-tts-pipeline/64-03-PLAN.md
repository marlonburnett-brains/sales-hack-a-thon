  <name>Task 1: Create TTS orchestrator CLI script</name>

  <action>
1. **Create the TTS orchestrator** at `scripts/tts.ts`:

   **CLI argument parsing** (no external deps, use process.argv):
   - Positional arg 1: tutorial name (required) -- e.g., "getting-started"
   - `--engine kokoro|chatterbox` (default: "kokoro")
   - `--step step-003` (optional, regenerate single step)
   - Print usage and exit if no tutorial name provided:
     "Usage: pnpm --filter tutorials tts <tutorial-name> [--engine kokoro|chatterbox] [--step step-003]"

   **Startup checks:**
   - Check ffmpeg is available: spawn `ffmpeg -version`, if fails throw "ffmpeg not found. Install with: brew install ffmpeg"
   - Load and validate tutorial script from `fixtures/{tutorialName}/script.json` using TutorialScriptSchema.parse (same pattern as capture.ts)

   **Engine instantiation:**
   - If engine is "kokoro": `new KokoroEngine()`
   - If engine is "chatterbox": `new ChatterboxEngine(path.join(process.cwd(), "assets", "reference-voice.wav"))`
   - Import both from their respective modules

   **Step filtering:**
   - If `--step` flag provided, filter to just that step. If step ID not found in script, print error and exit.
   - Otherwise, process all steps.

   **Per-step generation loop:**
   - For each step:
     a. Compute output path: `audio/{tutorialName}/step-{NNN}.wav` (use step.id which is already in "step-001" format)
     b. Ensure output directory exists (`fs.mkdirSync` recursive)
     c. Log: `Generating audio for ${step.id}: "${step.narration.substring(0, 50)}..."`
     d. Generate raw WAV: `const result = await engine.generate(step.narration, rawPath, { emotion: step.emotion })`
        - Use a temp path for raw output: `audio/{tutorialName}/.raw-{step.id}.wav`
     e. Post-process (normalize + silence): `await postProcessAudio(rawPath, finalPath)`
     f. Delete raw temp file
     g. Get final duration: `getWavDurationMs(finalPath)`
     h. Build TimingEntry: `{ stepId: step.id, audioFile: `${step.id}.wav`, durationMs, narration: step.narration, wordCount: step.narration.split(/\s+/).length }`

   **Manifest writing:**
   - If single-step mode (`--step`): call `updateManifestEntry(manifestPath, stepId, entry, engine)`
   - If full-tutorial mode: build complete TimingManifest and call `writeManifest(manifestPath, manifest)`
   - Manifest path: `audio/{tutorialName}/timing.json`

   **Summary output:**
   ```
   --- TTS Summary ---
   Tutorial: {title}
   Engine: {engine}
   Steps: {count}
   Total duration: {totalMs}ms ({seconds}s)
   Output: audio/{tutorialName}/
   ```

   **Error handling:**
   - Wrap the main loop in try/catch. On error, print the step that failed and the error message.
   - If a single step fails, continue to next steps (log warning) unless it's single-step mode.

2. **Create reference voice placeholder** at `assets/reference-voice.txt`:
   - Content: "Place reference-voice.wav here (~10s female narrator clip for Chatterbox voice consistency). Generate with: pnpm --filter tutorials tts getting-started --engine kokoro, then extract a clean 10s clip from the output."
   - This is a placeholder -- the actual .wav will be generated during first Kokoro run and manually curated.

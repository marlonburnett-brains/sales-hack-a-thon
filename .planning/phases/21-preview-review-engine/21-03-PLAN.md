**Step 2: Create the Slide Library page.**

**page.tsx** — Server Component:
- Fetch all templates via `listTemplatesAction()`
- For each template that has slides (slideCount > 0), fetch slides via `listSlidesAction(template.id)` in parallel using `Promise.all`
- Fetch thumbnails for each template via `getSlideThumbnailsAction(template.id)` in parallel
- Merge all slides into a flat array, annotating each with `templateName` and `templateId`
- Pass merged data to `<SlideLibraryClient>`
- If no slides at all, show empty state: "No ingested slides yet. Add and ingest templates first."

**slide-library-client.tsx** — "use client" interactive page:

State:
- `filter`: "all" | "unreviewed" | "approved" | "needs_correction" (default: "all")
- `page`: current page number (default: 0)
- `similarResults`: SimilarSlide[] | null (null = not searching)
- `searchingSlideId`: string | null (which slide's similarity is being searched)
- `isSearching`: boolean

Layout:
- Page title: "Slide Library" with subtitle "Browse and search all ingested slides"
- Filter bar: 4 buttons (All, Unreviewed, Approved, Needs Correction) styled as pill toggles
  - Active filter: bg-slate-900 text-white
  - Inactive: bg-slate-100 text-slate-700 hover:bg-slate-200
  - Show count per filter category
- Slide grid below: 4 columns on lg, 3 on md, 2 on sm, 1 on mobile
- Each slide card:
  - Thumbnail image (from thumbnails map, ~200px wide)
  - Template name as subtitle text
  - Slide index label ("Slide 3")
  - Review status badge (use Badge component: green for approved, amber for unreviewed, red for needs_correction)
  - Confidence score if available ("85%")
  - Primary classification tags (industry, pillar) as small chips
  - "Find Similar" button (Search icon from lucide-react)
  - Click card to navigate to `/templates/${templateId}/slides` (linking back to per-template viewer)
- Pagination: "Previous" / "Next" buttons with "Page X of Y" label
  - 20 slides per page
  - Disabled state when at first/last page

When "Find Similar" is clicked:
- Set `searchingSlideId` and `isSearching = true`
- Call `findSimilarSlidesAction(slideId, 8)`
- Display results in a `<SimilarityResults>` overlay/panel (see Task 2)
- Set `isSearching = false` when results arrive

For the grid, filter slides client-side based on `filter` state (all data is already loaded).
Sort by template name then slideIndex by default.

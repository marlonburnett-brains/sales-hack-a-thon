<objective>
Build the modification planner -- an LLM-driven agent that examines a slide's element map and deal context to produce a surgical ModificationPlan specifying which text elements to change and what the new content should be.

Purpose: This is the intelligence layer that makes deck modifications context-aware rather than template-driven. Instead of global find-and-replace with placeholder tokens, each slide gets a tailored modification plan based on its actual elements and the deal's specific context.

Output: A registered "modification-planner" named agent and a `planSlideModifications()` function that downstream Phase 55 (Modification Executor) will call per-slide.
</objective>

Create the core modification planner module at `apps/agent/src/generation/modification-planner.ts`. This is the main deliverable of Phase 53.

**Exports:**
- `planSlideModifications(params: PlanModificationsParams): Promise<PlanModificationsResult>` -- main entry point
- `PlanModificationsParams` interface
- `PlanModificationsResult` interface

**Interface definitions:**

```typescript
interface PlanModificationsParams {
  slideId: string;           // SlideEmbedding database ID
  slideObjectId: string;     // Google Slides page object ID (from assembled presentation, not DB)
  dealContext: DealContext;   // from @lumenalta/schemas
}

interface PlanModificationsResult {
  plan: ModificationPlan;    // from ./modification-plan-schema
  usedFallback: boolean;     // true if no element maps found, downstream should use replaceAllText
}
```

**Implementation steps for `planSlideModifications`:**

1. **Load elements**: Query `prisma.slideElement.findMany({ where: { slideId: params.slideId } })` to get all SlideElement records for this slide.

2. **Filter to text-bearing elements** (FR-5.3): Keep only elements where `(elementType === "text" || elementType === "shape") && contentText.trim().length > 0`. This excludes images, tables, groups, and empty shapes.

3. **Fallback check** (FR-5.6): If filtered elements array is empty, return a placeholder fallback plan:
   ```typescript
   {
     plan: { slideId: params.slideId, slideObjectId: params.slideObjectId, modifications: [], unmodifiedElements: [] },
     usedFallback: true,
   }
   ```
   The empty modifications + `usedFallback: true` signals downstream (Phase 55) to use `replaceAllText` placeholder injection instead.

4. **Build prompt** (FR-5.2, FR-5.4): Create a detailed prompt with:
   - Deal context block (companyName, industry, pillars, persona, funnelStage)
   - Slide identification (slideId, slideObjectId) -- these MUST be passed through to the LLM so it returns them in the structured output
   - Element map listing each text-bearing element with elementId, elementType, and contentText (truncate contentText > 500 chars with "... [truncated]")
   - Modification rules that explicitly distinguish:
     - **MODIFY** (deal-specific): company names, industry references, persona mentions, generic summary bullets that should reference the target company, placeholder-like content
     - **PRESERVE** (structural): methodology descriptions, capability definitions, case study specifics, process step labels, section headers that define the slide's purpose, numbered lists describing capabilities, specific client success stories
   - Instruction that new content must be same length or shorter than current content
   - Instruction that every element must appear in either modifications[] or unmodifiedElements[]

5. **Call LLM**: Use `executeRuntimeProviderNamedAgent` with:
   ```typescript
   const result = await executeRuntimeProviderNamedAgent({
     agentId: "modification-planner",
     messages: [{ role: "user", content: prompt }],
     options: createJsonResponseOptions(MODIFICATION_PLAN_SCHEMA as Record<string, unknown>),
   });
   ```

6. **Parse response**: `JSON.parse(result.text) as ModificationPlan`

7. **Post-validate** (pitfall mitigation): Build a `Set<string>` of known element IDs from the filtered elements. Strip any modifications targeting unknown elementIds (LLM hallucination guard). Strip any unmodifiedElements entries with unknown IDs. Log warnings for stripped entries using `console.warn("[modification-planner] ...")`.

8. **Return**: `{ plan: validatedPlan, usedFallback: false }`

**Error handling**: Wrap the LLM call in try/catch. On failure, log the error and return the fallback plan with `usedFallback: true` so generation continues gracefully.

**Imports needed:**
- `import type { DealContext } from "@lumenalta/schemas"` (shared type)
- `import { type ModificationPlan, MODIFICATION_PLAN_SCHEMA } from "./modification-plan-schema"` (existing from Phase 50)
- `import { createJsonResponseOptions, executeRuntimeProviderNamedAgent } from "../lib/agent-executor"` (existing executor)
- `import { prisma } from "../lib/db"` (existing Prisma client)

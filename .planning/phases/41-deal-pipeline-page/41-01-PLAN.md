1. **Extend Deal model in schema.prisma** -- add these fields after `driveFolderId`:
   - `status String @default("open")` -- values: "open", "won", "lost", "abandoned"
   - `ownerId String?` -- Supabase user ID or null
   - `ownerEmail String?` -- display email
   - `ownerName String?` -- display name
   - `collaborators String @default("[]")` -- JSON array of `[{id?, email, name?}]`
   - Add indexes: `@@index([status])` and `@@index([ownerId])`

2. **Create migration** using `prisma migrate dev --name add-deal-pipeline-fields` from `apps/agent/` directory. Use `--create-only` first to inspect the SQL. Verify the migration SQL includes `DEFAULT 'open'` for status and `DEFAULT '[]'` for collaborators so existing records get proper defaults. Then apply.

   CRITICAL: Follow CLAUDE.md -- NEVER use `prisma db push`. NEVER use `prisma migrate reset`.

3. **Modify the existing GET /deals route** in index.ts to accept query params for filtering:
   - `?status=open` (or won/lost/abandoned/all) -- filter by status. Default behavior: if no status param, return all (backward compatible)
   - `?assignee=me&userId=xxx` -- filter where ownerId equals userId OR collaborators string contains userId
   - `?assignee={specificUserId}` -- filter where ownerId equals that value OR collaborators contains it
   - Keep existing include of company and interactions with brief

4. **Add PATCH /deals/:id/status** route after existing deal routes:
   - Accept `{ status: string }` body
   - Validate status is one of: "open", "won", "lost", "abandoned" -- return 400 if invalid
   - Update the deal, include company in response
   - Return updated deal JSON

5. **Add PATCH /deals/:id/assignment** route:
   - Accept `{ ownerId?, ownerEmail?, ownerName?, collaborators? }` body
   - Update the deal. For collaborators, `JSON.stringify(collaborators ?? [])` before storing
   - Include company in response
   - Return updated deal JSON

6. **Add GET /users/known** route:
   - Query `prisma.userGoogleToken.findMany({ where: { isValid: true }, select: { userId: true, email: true }, orderBy: { lastUsedAt: "desc" } })`
   - Map to `{ id: t.userId, email: t.email, name: t.email.split("@")[0].replace(/[.-]/g, " ").replace(/\b\w/g, c => c.toUpperCase()) }` -- extract display name from email by splitting on @, replacing dots/hyphens with spaces, title-casing
   - Return JSON array

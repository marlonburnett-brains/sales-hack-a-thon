   - Define ActionSchema as a Zod discriminated union with types: "click" (selector), "fill" (selector, value), "select" (selector, value), "wait" (selector), "hover" (selector), "keyboard" (key)
   - Define StepSchema with fields: id (string, e.g. "step-001"), url (string), narration (string for conversational narration text), actions (optional array of ActionSchema), waitFor (optional string selector), zoomTarget (optional object with selector + scale defaulting to 1.5), mockOverrides (optional record for step-specific fixture overrides)
   - Define TutorialScriptSchema with fields: id (string, e.g. "getting-started"), title (string), description (string), steps (array of StepSchema, min 1), fixtures (optional string for fixture set name)
   - Export all schemas AND inferred types: TutorialScript, TutorialStep, TutorialAction
   - Use the narration style from CONTEXT: conversational guide tone

2. Create `apps/tutorials/fixtures/types.ts`:
   - Define Zod schemas that mirror the API response shapes from apps/web/src/lib/api-client.ts
   - Read api-client.ts to understand return types for: companies, deals, users, templates, slides, settings, agent-prompts, deck-structures, interactions, generation-logs, tokens/check
   - Create schemas: CompanyFixtureSchema, DealFixtureSchema, UserFixtureSchema, TemplateFixtureSchema, SlideFixtureSchema, DriveSettingsFixtureSchema, etc.
   - These are validation schemas for fixtures -- they ensure fixture data matches what the real API returns

3. Create `apps/tutorials/fixtures/factories.ts`:
   - Import Zod schemas from ./types.ts
   - Define TUTORIAL_COMPANY constant: "Meridian Dynamics", Technology Consulting industry (per RESEARCH.md example)
   - Define TUTORIAL_USER: a consistent test user with plausible name/email
   - Factory functions: createCompanyFixture(overrides?), createDealFixture(overrides?), createUserFixture(overrides?), createTemplateFixture(overrides?), createSlideFixture(overrides?)
   - Each factory merges defaults with overrides, validates against Zod schema, returns validated data
   - Create a createSharedFixtures() function that generates the full shared fixture set (users, companies, deals) with production-realistic data -- plausible company names, realistic deal amounts ($50K-$500K range), real-sounding user names
   - All dates use ISO 8601 format, all IDs use UUID-like format

4. Create `apps/tutorials/fixtures/loader.ts`:
   - Export loadFixtures(tutorialName: string) function
   - Loads shared fixtures from fixtures/shared/*.json
   - Loads tutorial-specific overrides from fixtures/{tutorialName}/overrides.json if it exists
   - Deep merges overrides onto shared fixtures
   - Returns a typed FixtureSet object with all fixture categories

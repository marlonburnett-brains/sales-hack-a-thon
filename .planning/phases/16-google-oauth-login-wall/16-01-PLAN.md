1. Create `apps/web/src/middleware.ts` (MUST be at src/ root, NOT inside app/):
   - Use pattern from 16-RESEARCH.md Pattern 3 exactly
   - Create Supabase server client with request cookie proxy (getAll from request.cookies, setAll updates both request and response)
   - Call `supabase.auth.getUser()` -- this BOTH refreshes tokens AND checks auth status. NEVER remove this call.
   - If no user AND path is not `/login` or `/auth/*`, redirect to `/login?next={returnPath}`
   - Preserve the full return URL (pathname + search params) in the `next` query parameter
   - Matcher config: `'/((?!_next/static|_next/image|favicon.ico|.*\\.(?:svg|png|jpg|jpeg|gif|webp)$).*)'`
   - If user IS authenticated and hits `/login`, redirect to `/deals` (prevent login page when already logged in)

2. Create `apps/web/src/app/auth/callback/route.ts` (OAuth callback):
   - Export GET handler
   - Extract `code` and `next` (default `/deals`) from searchParams
   - Call `supabase.auth.exchangeCodeForSession(code)` to complete PKCE flow
   - After successful exchange, call `supabase.auth.getUser()` (NOT getSession)
   - SERVER-SIDE domain enforcement: check `user.email.endsWith('@lumenalta.com')`. If NOT lumenalta.com:
     - Call `supabase.auth.signOut()` to clear the invalid session
     - Redirect to `/login?error=domain`
   - If valid domain: redirect to `next` URL (which preserves the original destination)
   - On any auth error: redirect to `/login?error=auth`

3. Create `apps/web/src/lib/actions/auth.ts`:
   - 'use server' directive
   - Export async `signOut()` function
   - Creates server Supabase client, calls `supabase.auth.signOut()`, then `redirect('/login')`
   - Import redirect from 'next/navigation', createClient from '@/lib/supabase/server'

4. Restructure routes into (authenticated) route group:
   - Create `apps/web/src/app/(authenticated)/` directory
   - Move ALL of `apps/web/src/app/deals/` into `apps/web/src/app/(authenticated)/deals/`
   - Move ALL of `apps/web/src/app/api/` into `apps/web/src/app/(authenticated)/api/`
   - IMPORTANT: Route groups are URL-transparent -- `/deals` and `/api/upload` URLs remain unchanged
   - After moving, verify all imports in moved files still resolve correctly

5. Create `apps/web/src/app/(authenticated)/layout.tsx`:

   - For now, render a minimal nav bar with just the Briefcase icon + "Lumenalta Sales" branding (same as current root layout nav)

   - Plan 02 will enhance this with UserNav component -- for now just ensure the nav bar renders
   - This is a Server Component -- it can await createClient() from supabase/server to get user data

6. Update `apps/web/src/app/layout.tsx` (root layout):

   - Keep all imports for font, cn, globals.css, metadata

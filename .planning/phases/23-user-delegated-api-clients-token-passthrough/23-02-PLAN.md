<objective>
Create the web-side token passthrough pipeline: extract Google tokens from Supabase session, send them as custom headers via fetchWithGoogleAuth, and wire all Google-triggering Server Actions to use the new wrapper.

Purpose: Completes the end-to-end token passthrough so interactive requests carry user credentials from browser through Server Actions to the agent, enabling user-delegated Google API access.

Output: New `google-token.ts` helper, `fetchWithGoogleAuth` in api-client.ts, and updated Server Actions for templates, slides, and touch workflows.
</objective>

**1. Create `apps/web/src/lib/supabase/google-token.ts`:**

- Import `createClient` from `./server`
- Export `async function getGoogleAccessToken(): Promise<{ accessToken: string | null; userId: string | null }>`
  - `const supabase = await createClient()`
  - `const { data: { session } } = await supabase.auth.getSession()` -- provider_token is only available if session is fresh from login
  - `const { data: { user } } = await supabase.auth.getUser()` -- reliable userId
  - Return `{ accessToken: session?.provider_token ?? null, userId: user?.id ?? null }`

**2. Add `fetchWithGoogleAuth` to `apps/web/src/lib/api-client.ts`:**
- Add AFTER the existing `fetchJSON` function (keep fetchJSON unchanged)
- Import `getGoogleAccessToken` from `@/lib/supabase/google-token` at the top of the file (static import, both are server-only)
- Export new function:
```typescript
export async function fetchWithGoogleAuth<T>(
  path: string,
  init?: RequestInit,
): Promise<T> {
  const { accessToken, userId } = await getGoogleAccessToken();

  const googleHeaders: Record<string, string> = {};
  if (accessToken) googleHeaders['X-Google-Access-Token'] = accessToken;
  if (userId) googleHeaders['X-User-Id'] = userId;

  return fetchJSON<T>(path, {
    ...init,
    headers: {
      ...init?.headers,
      ...googleHeaders,
    },
  });
}
```

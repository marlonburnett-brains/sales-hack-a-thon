### Integration Issues

**1. Touch 4 Standalone Brief Review — workflowRunId null**

`brief-review-client.tsx` line 71 sends `runId: reviewData.brief.workflowRunId ?? ""`. The `workflowRunId` field on the Brief record is null until the approve endpoint itself sets it (chicken-and-egg). The inline `touch-4-form.tsx` path works because it holds `runId` in React state. The standalone page has no way to supply the correct Mastra `runId`.

**2. Pre-Call Missing FeedbackSignal (minor)**

The pre-call workflow creates an `InteractionRecord` with status "approved" but creates no `FeedbackSignal`. All other touch flows (1, 2, 3, 4) create FeedbackSignals. This is a minor consistency gap, not a requirement violation.

**3. Dead Schema Exports (minor)**

`InteractionRecordSchema` and `FeedbackSignalSchema` are exported from `@lumenalta/schemas` but imported by neither app. Both apps define local types instead.

## Root Cause Analysis

All four unsatisfied/partial requirements (CONT-01, CONT-02, CONT-03, CONT-04) share a single root cause: **Google Drive shortcut target access**. The service account has access to the Hack-a-thon folder but not to the Shared Drives that the shortcuts point to. This is a configuration gap, not a code gap — the ingestion pipeline is production-ready and processes all accessible content correctly.

**Remediation path:**
1. Grant service account Viewer access on each target Shared Drive (14 sources)
2. Re-run `npx tsx --env-file=.env src/ingestion/run-ingestion.ts`
3. Verify `coverage-report.json` shows `caseStudies > 0` for target industries
4. Update REQUIREMENTS.md traceability table to reflect actual status

1. Define `PooledAtlusAuthResult` interface:
   ```typescript
   export interface PooledAtlusAuthResult {
     token: string;
     source: 'pool' | 'env';
     userId?: string;
   }
   ```

2. Implement `getPooledAtlusAuth(): Promise<PooledAtlusAuthResult | null>` by cloning the pattern from `getPooledGoogleAuth()` in `apps/agent/src/lib/google-auth.ts` lines 92-174. Key differences from Google version:
   - No OAuth2Client or refresh token flow -- just decrypt and return the raw token
   - Source is `'pool' | 'env'` (not `'pool' | 'service_account'`)
   - Fallback is `process.env.ATLUS_API_TOKEN` (not service account)
   - Do NOT create ActionRequired on failure here (that's Plan 27-03's job)

   Implementation:
   - `findMany({ where: { isValid: true }, orderBy: { lastUsedAt: 'desc' } })`
   - For-loop with try/catch per token
   - On success: fire-and-forget `update({ where: { id }, data: { lastUsedAt: new Date() } }).catch(() => {})`
   - On failure: fire-and-forget `update({ where: { id }, data: { isValid: false, revokedAt: new Date() } }).catch(() => {})`
   - After loop: check pool health `count({ where: { isValid: true } })` -- if < 3, `console.warn('[atlus-pool] WARNING: Only N valid token(s) remaining')`
   - After loop: try `process.env.ATLUS_API_TOKEN` fallback
   - Return null if everything exhausted

3. Create test file at `apps/agent/src/lib/__tests__/atlus-auth.test.ts`. Mock `prisma` (use `vi.mock('../db')`) and `token-encryption` (use `vi.mock('../token-encryption')`). Test all 7 behaviors listed above. Use `vi.spyOn(console, 'warn')` for pool health warning test.

- Returns `{ client_id: string } | null`

**Create `apps/agent/src/lib/mcp-client.ts`:**

This is the core singleton module. Follow the architecture pattern from RESEARCH.md Pattern 1, with these additions per user decisions:

**Module state:**
- `client: MCPClient | null` -- current client instance
- `createdAt: number` -- timestamp of client creation
- `currentAuth: { token: string; refreshToken?: string; userId?: string; source: 'pool' | 'env' } | null` -- current auth context
- `fallbackMode: boolean` -- set true if MCP is down (pre-set on boot failure)
- `cachedExtractionPrompt: string | null` -- adaptive LLM prompt cache (resets on recycle)
- `refreshPromise: Promise<boolean> | null` -- mutex for token refresh (Pitfall 2)
- `cachedClientId: string | null` -- cached dynamic client_id for token refresh

**Exports:**

1. `initMcp(): Promise<void>` -- Eager connection on agent boot:
   - If `ATLUS_USE_MCP === 'false'`, log and return (no-op)
   - Call `getPooledAtlusAuth()` to get initial token
   - If no token available, set `fallbackMode = true`, log, return
   - Call `registerAtlusClient()` to get/cache a client_id for future refresh
   - Create MCPClient via `createClient()` helper
   - Run health check via `listTools()` with 5s timeout
   - If health check fails: set `fallbackMode = true`, log warning, disconnect client, set client to null
   - If health check succeeds: log tool count, set `fallbackMode = false`
   - Catch all errors -- NEVER throw from initMcp (agent boot must not fail)

2. `getMcpClient(): Promise<MCPClient | null>` -- Get client with lazy recycle:
   - If `ATLUS_USE_MCP === 'false'` or `fallbackMode`, return null
   - If client exists and age > `ATLUS_MCP_MAX_LIFETIME_MS`: disconnect, set client = null, clear `cachedExtractionPrompt`
   - If no client: get fresh auth from pool, create new client, set createdAt
   - Return client (or null if creation failed)

3. `callMcpTool(toolName: string, args: Record<string, unknown>): Promise<unknown>` -- Call a tool with retry/rotate:
   - Get client via `getMcpClient()`
   - If null, throw `Error('MCP not available')`
   - Get tools via `client.listTools()`
   - Look up tool by namespaced name: `atlus_${toolName}` (per Pitfall 3)
   - Call `tool.execute(args)`
   - On success: return result
   - On error that looks like 401/auth failure: attempt token refresh via `handleAuthFailure()`
   - `handleAuthFailure()`:
     - Uses `refreshPromise` mutex to serialize concurrent refresh attempts (Pitfall 2)
     - If `currentAuth.refreshToken` exists: call `refreshAtlusToken(refreshToken, cachedClientId)`
     - On refresh success: call `updateAtlusTokenInDb()`, update `currentAuth.token`, disconnect + recreate client, retry tool call once
     - On refresh failure: mark token invalid in DB (`prisma.userAtlusToken.update isValid: false`), call `getPooledAtlusAuth()` for next token, recreate client, retry once
     - If no more tokens: set `fallbackMode = true`, throw
   - On non-auth error: throw (let caller handle)

4. `isMcpAvailable(): boolean` -- Quick sync check (returns `!fallbackMode && ATLUS_USE_MCP !== 'false'`)

5. `shutdownMcp(): Promise<void>` -- Graceful shutdown:
   - If client exists: `await client.disconnect()`
   - Set client = null
   - Log "[mcp] Disconnected cleanly"

6. `getCachedExtractionPrompt(): string | null` and `setCachedExtractionPrompt(prompt: string): void` -- Getter/setter for the adaptive LLM prompt cache (used by search adapter in Plan 02)

**Internal helper `createClient()`:**
- Creates new MCPClient with config per RESEARCH.md Pattern 1
- `id: "atlus-mcp"`
- `servers.atlus.url: new URL("https://knowledge-base-api.lumenalta.com/sse")`
- `servers.atlus.timeout: 30_000`
- `servers.atlus.connectTimeout: 5_000` (per Pitfall 4)
- `servers.atlus.fetch`: thin callback that injects `Authorization: Bearer ${currentAuth.token}` -- does NOT do refresh logic (per user decision: fetch callback stays thin)
- `timeout: 30_000`

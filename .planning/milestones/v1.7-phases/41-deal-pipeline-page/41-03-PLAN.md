1. **Create deal-status-action.tsx** -- "use client" component:
   - Props: `dealId: string`, `currentStatus: string`, `onStatusChange?: (newStatus: string) => void`
   - Uses shadcn/ui DropdownMenu for the trigger and menu items
   - Trigger: a small button showing current status with a ChevronDown icon. Use the same status color scheme as status badges. Render as a compact pill-shaped button.
   - Menu items: Open, Won, Lost, Abandoned -- each with a colored dot indicator
   - Hide the current status from the menu (no need to "change" to the same status)
   - For terminal statuses (Won, Lost, Abandoned): show an AlertDialog confirmation before applying
     - Title: "Change status to {Status}?"
     - Description: "This marks the deal as {status}. You can change it back later."
     - Cancel and Confirm buttons
   - For Open status transitions: apply immediately (no confirmation needed)
   - On confirm/apply: call `updateDealStatusAction(dealId, newStatus)`, show toast via sonner ("Status updated to {Status}"), call `onStatusChange` callback if provided, then `router.refresh()` to re-fetch data
   - IMPORTANT: Must stopPropagation on DropdownMenu trigger click to prevent the parent Link on deal cards from navigating. Use `e.preventDefault(); e.stopPropagation();` on the trigger's onClick.
   - Show loading spinner (Loader2) while the update is in flight. Disable the trigger during update.

2. **Create deal-assignment-picker.tsx** -- "use client" component:
   - Props: `dealId: string`, `currentOwnerId?: string | null`, `currentOwnerEmail?: string | null`, `currentOwnerName?: string | null`, `currentCollaborators: Array<{id?: string; email: string; name?: string}>`, `knownUsers: KnownUser[]`, `onAssignmentChange?: () => void`
   - Uses shadcn/ui Popover with a trigger button
   - Trigger button: shows owner name/email or "Assign owner" placeholder. Small, subtle styling.
   - Popover content has two sections:
     a. **Owner** section:
        - Searchable list of known users (filter by typing)
        - Option to type a freeform @lumenalta.com email if no match
        - Clicking a user sets them as owner
        - Input field at top for filtering/freeform entry
        - Validate freeform entries end with @lumenalta.com
     b. **Collaborators** section:
        - Same hybrid picker but multi-select (checkboxes next to users)
        - Currently selected collaborators shown with X buttons to remove
        - Same freeform email entry option
   - On any change (owner or collaborator): call `updateDealAssignmentAction(dealId, { ownerId, ownerEmail, ownerName, collaborators })`, show toast "Assignment updated", call `onAssignmentChange` callback, then `router.refresh()`
   - IMPORTANT: Must stopPropagation on Popover trigger click (same Link prevention as status action)

3. **Create deal-assignee-filter.tsx** -- "use client" component:
   - Props: `currentAssignee: string`, `knownUsers: KnownUser[]`
   - Uses `useRouter`, `useSearchParams`, `usePathname`
   - Renders a Select dropdown (shadcn/ui) with options:
     - "All deals" (value: "all")
     - "Assigned to me" (value: "me")
     - Divider, then each known user by name/email (value: their userId)
   - On change: update `assignee` param in URL via router.push, preserving other params
   - Compact styling to fit alongside the status pills: `text-xs` font size, minimal padding

**First, install no new packages for this task.** The chat uses native fetch streaming.

**Agent-side chat route** -- Add to `apps/agent/src/mastra/index.ts`:
`POST /agent-configs/:agentId/chat` -- Accept `{ message: string, currentPrompt: string }`. Load the agent's published config. Build a system prompt that instructs the AI: "You are a prompt engineering assistant. The user wants to improve an agent's system prompt. The agent's current role prompt is provided. When you suggest changes, output the COMPLETE updated prompt between delimiters `---PROMPT_UPDATE---` and `---END_PROMPT_UPDATE---`. Outside the delimiters, explain your reasoning." Use the Mastra agent's `generate` (streaming) with the system prompt + user message + current prompt as context. Stream the response.

**Streaming proxy** -- `apps/web/src/app/api/agents/chat/route.ts`:
Follow the deck-structures/chat/route.ts pattern exactly. Validate body with zod: `{ agentId: string, message: string, currentPrompt: string }`. Proxy to `${AGENT_SERVICE_URL}/agent-configs/${agentId}/chat`. Pipe streaming response back.

**agent-chat-panel.tsx** -- "use client" component. Props: `{ agentId: string, currentPrompt: string, onPromptUpdate: (newPrompt: string) => void }`.

Layout (persistent bottom panel, similar to ChatGPT input area):
1. Fixed to bottom of the detail page content area (not viewport-fixed, but sticky at bottom of the scrollable content). Use `sticky bottom-0` with white bg and top border.
2. Message history area above input (scrollable, max-h-64). Each message: user messages right-aligned with blue bg, assistant messages left-aligned with gray bg. Render markdown-like formatting.
3. Input row: text input + Send button (lucide Send icon). Disabled while streaming.
4. Apply mode toggle: Small toggle in the panel header. Two states: "Auto-apply" (changes apply to textarea immediately) and "Review first" (shows diff with Apply/Dismiss buttons). Store preference in `localStorage` key `agent-chat-apply-mode`. Default: "review".

Streaming behavior:
- On send: POST to `/api/agents/chat` with `{ agentId, message, currentPrompt }`.
- Read streaming response using `ReadableStream` reader (same pattern as ChatBar).
- Parse response for `---PROMPT_UPDATE---` delimiter. When found, extract the text between delimiters.
- **Auto-apply mode:** Call `onPromptUpdate(extractedPrompt)` immediately.
- **Review mode:** Show a diff overlay inline in the chat (green additions, red removals using simple line comparison). Show "Apply" and "Dismiss" buttons. Apply calls `onPromptUpdate(extractedPrompt)`. Dismiss hides the diff.

Chat state is ephemeral (client-side only, per user decision discretion -- no DB persistence).

         d) Show initial toast per item: `toast("Queued for ingestion", { id: \`ingest-${doc.slideId}\` })` -- use stable toast ID per user decision
         e) Fire API call (already done)
         f) On error: rollback status AND show error toast replacing the queued one: `toast.error("Failed to start ingestion", { id: \`ingest-${doc.slideId}\` })`

    3. **Progress toast updates:**
       - Modify `startPolling` to update toast progress using the same stable toast ID:

         - When item errors: `toast.error("Ingestion failed", { id: \`ingest-${item.id}\` })`

       - If Discovery cards have a dropdown menu with "Ingest" option, ensure:

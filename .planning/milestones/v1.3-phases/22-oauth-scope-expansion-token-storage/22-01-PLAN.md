<objective>
Create the agent-side infrastructure for encrypted token storage: the UserGoogleToken Prisma model, AES-256-GCM encryption module with tests, the GOOGLE_TOKEN_ENCRYPTION_KEY env var, and the POST /tokens + GET /tokens/check/:userId API routes.

Purpose: The web auth callback (Plan 02) will call POST /tokens to store refresh tokens. The middleware re-consent check (Plan 02) will call GET /tokens/check/:userId. This plan creates both endpoints and the underlying encryption + storage layer.

Output: Working agent API endpoints for token storage and checking, backed by an encrypted database model with unit-tested encryption.
</objective>

  <name>Task 1: Token encryption module + UserGoogleToken model + migration</name>
  <files>apps/agent/src/lib/token-encryption.ts, apps/agent/src/lib/__tests__/token-encryption.test.ts, apps/agent/prisma/schema.prisma, apps/agent/src/env.ts</files>
  <behavior>
    - encryptToken("test-refresh-token") returns { encrypted, iv, authTag } where all are non-empty base64 strings
    - decryptToken(encrypted, iv, authTag) returns the original plaintext "test-refresh-token"
    - encryptToken produces different iv and encrypted output on each call (randomized IV)
    - getEncryptionKey() throws if GOOGLE_TOKEN_ENCRYPTION_KEY is not set
    - getEncryptionKey() throws if key is not exactly 64 hex chars (32 bytes)
    - Tampered authTag causes decryptToken to throw
    - Tampered encrypted data causes decryptToken to throw
  </behavior>
  <action>
    1. Add GOOGLE_TOKEN_ENCRYPTION_KEY to apps/agent/src/env.ts as `z.string().length(64)` (64 hex chars = 32 bytes). Make it optional with `.optional()` so the server still starts without it (graceful degradation for existing deploys); the encryption module itself validates at call time.

    2. Create apps/agent/src/lib/token-encryption.ts with:
       - `getEncryptionKey()`: reads GOOGLE_TOKEN_ENCRYPTION_KEY from process.env, validates 64 hex chars, returns Buffer.from(key, 'hex')
       - `encryptToken(plaintext: string)`: uses createCipheriv('aes-256-gcm', key, randomBytes(16)), returns { encrypted: string, iv: string, authTag: string } all base64-encoded
       - `decryptToken(encrypted: string, iv: string, authTag: string)`: uses createDecipheriv, sets authTag BEFORE update/final, returns plaintext string
       - Use exact pattern from 22-RESEARCH.md Pattern 1

    3. Create apps/agent/src/lib/__tests__/token-encryption.test.ts with vitest tests covering all behaviors above. Set GOOGLE_TOKEN_ENCRYPTION_KEY in test setup using a test key generated from `crypto.randomBytes(32).toString('hex')`.

    4. Add UserGoogleToken model to apps/agent/prisma/schema.prisma:
       ```prisma
       model UserGoogleToken {
         id               String    @id @default(cuid())
         userId           String    @unique
         email            String
         encryptedRefresh String
         iv               String
         authTag          String
         lastUsedAt       DateTime  @default(now())
         isValid          Boolean   @default(true)
         revokedAt        DateTime?
         createdAt        DateTime  @default(now())
         updatedAt        DateTime  @updatedAt

         @@index([isValid, lastUsedAt])
         @@index([email])
       }
       ```

    5. Run migration: `cd apps/agent && npx prisma migrate dev --name add-user-google-token`. Per CLAUDE.md: NEVER use `prisma db push` or `prisma migrate reset`.

    6. Run tests: `cd apps/agent && npx vitest run src/lib/__tests__/token-encryption.test.ts`

  <files>apps/agent/src/mastra/index.ts</files>
  <action>
    1. Import encryptToken from "../lib/token-encryption" at the top of mastra/index.ts.

    2. Add POST /tokens route using registerApiRoute pattern (follows existing routes in the file). Handler:
       - Parse body with zod: `z.object({ userId: z.string().min(1), email: z.string().email(), refreshToken: z.string().min(1) })`
       - Call `encryptToken(data.refreshToken)` to get { encrypted, iv, authTag }
       - Upsert into prisma.userGoogleToken:
         - where: { userId: data.userId }
         - update: { encryptedRefresh: encrypted, iv, authTag, email: data.email, isValid: true, revokedAt: null, lastUsedAt: new Date() }
         - create: { userId: data.userId, email: data.email, encryptedRefresh: encrypted, iv, authTag }
       - Return `c.json({ success: true, tokenId: token.id })`
       - Wrap in try/catch; on zod error return 400, on encryption error return 500 with message

    3. Add GET /tokens/check/:userId route. Handler:
       - Extract userId from `c.req.param("userId")`
       - Query: `prisma.userGoogleToken.findUnique({ where: { userId }, select: { isValid: true } })`
       - Return `c.json({ hasToken: !!token?.isValid })`

    4. Both routes are protected by the existing SimpleAuth Bearer token middleware (AGENT_API_KEY) that already applies to all routes in mastra/index.ts.

    5. Verify by running the full agent test suite to ensure no regressions: `cd apps/agent && npx vitest run`

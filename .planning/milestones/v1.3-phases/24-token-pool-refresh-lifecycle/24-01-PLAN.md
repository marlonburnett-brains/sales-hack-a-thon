<objective>
Implement token pool system for background jobs and ActionRequired model for tracking manual user actions.

Purpose: Background jobs (staleness polling, ingestion queue) currently use the service account for all Google API calls. This plan adds `getPooledGoogleAuth()` which iterates through stored user refresh tokens ordered by `lastUsedAt DESC`, falling back to the service account when the pool is exhausted. When even the service account fails on a resource, an `ActionRequired` record is created so users can see what needs fixing. Token rotation from Google is captured via the `tokens` event listener.

Output: Working pooled auth system, ActionRequired Prisma model with migration, background jobs wired to pooled auth, ActionRequired CRUD API routes on agent.
</objective>

**2. Add `getPooledGoogleAuth()` to google-auth.ts:**

Add a new `PooledAuthResult` interface and `getPooledGoogleAuth()` function:

```typescript

  accessToken?: string;       // Access token for passing to existing GoogleAuthOptions pattern
  source: 'pool' | 'service_account';
  userId?: string;
}
```

Implementation:
- Query `prisma.userGoogleToken.findMany({ where: { isValid: true }, orderBy: { lastUsedAt: 'desc' } })`
- Iterate ALL valid tokens (per locked decision: try ALL, not capped at 5)
- For each token: decrypt with `decryptToken()`, create `OAuth2Client(env.GOOGLE_CLIENT_ID, env.GOOGLE_CLIENT_SECRET)`, set credentials with refresh_token
- **LIFE-01**: Attach `client.on('tokens', ...)` event listener. When `tokens.refresh_token` is present, encrypt with `encryptToken()` and update the DB record (fire-and-forget `.catch()`)
- Call `client.getAccessToken()` to validate. If no access token returned, throw
- **POOL-04**: On success, fire-and-forget update `lastUsedAt: new Date()` on the token record
- **POOL-03**: On failure (catch block), mark token `isValid: false, revokedAt: new Date()` (fire-and-forget)
- Return `{ accessToken, source: 'pool', userId: token.userId }`
- **POOL-05**: After the loop (whether successful pool hit or not), count valid tokens. If < 3: `console.warn('[token-pool] WARNING: Only ${validCount} valid token(s) remaining in pool')`
- If all tokens exhausted, fall back to service account: return `{ source: 'service_account' }` (no accessToken = callers use SA)

Import `PrismaClient`, `OAuth2Client`, `decryptToken`, `encryptToken`, `env` at top of file. Use a module-level `const prisma = new PrismaClient()`.

Keep the existing `getGoogleAuth()` (private, service account) and all existing exports unchanged.

      const userId = c.req.query("userId");

**2. Create `apps/agent/src/lib/token-cache.ts`:**
- In-memory `Map<string, CachedToken>` where `CachedToken = { accessToken: string; expiresAt: number }`
- `TOKEN_TTL_MS = 50 * 60 * 1000` (50 minutes, Google tokens last 60min)
- Export `async function getAccessTokenForUser(userId: string): Promise<string | null>`
  - Check cache first; if valid (expiresAt > Date.now()), return cached token
  - Otherwise: query `prisma.userGoogleToken.findUnique({ where: { userId, isValid: true } })`
  - If no record, return null
  - Call `decryptToken(record.encryptedRefresh, record.iv, record.authTag)` from `token-encryption.ts`
  - Create `new OAuth2Client(env.GOOGLE_CLIENT_ID, env.GOOGLE_CLIENT_SECRET)` from `google-auth-library`
  - Call `client.setCredentials({ refresh_token: decryptedRefresh })` then `client.getAccessToken()`
  - If token returned: cache it, update `lastUsedAt` on DB record, return token
  - If refresh fails: mark `isValid: false` and `revokedAt: new Date()` on DB record, delete cache entry, return null
- Use a per-userId promise dedup to prevent concurrent refresh races: if a refresh is in-flight for a userId, subsequent callers await the same promise
- Import shared Prisma instance (check how other files import it -- likely `@prisma/client` or a shared instance)
- Periodic cleanup: on each cache access, if cache.size > 100, sweep expired entries (simple, not a timer)

**3. Create `apps/agent/src/lib/request-auth.ts`:**
- Export `interface GoogleAuthResult { accessToken?: string; userId?: string; }`
- Export `async function extractGoogleAuth(c: Context): Promise<GoogleAuthResult>` where Context is from 'hono'
  - Read `X-Google-Access-Token` header from `c.req.header('X-Google-Access-Token')`
  - Read `X-User-Id` header from `c.req.header('X-User-Id')`
  - Priority chain per locked decision:
    1. If `accessToken` present: return `{ accessToken, userId: userId ?? undefined }`
    2. If `userId` present (no accessToken): call `getAccessTokenForUser(userId)` from token-cache.ts
       - If resolved: return `{ accessToken: resolved, userId }`
       - If null: return `{ userId }` (factory will fall back to service account)
    3. Neither present: return `{}` (service account fallback)

**4. Modify `apps/agent/src/lib/google-auth.ts`:**
- Add import for `OAuth2Client` from `google-auth-library`
- Export `interface GoogleAuthOptions { accessToken?: string; userId?: string; }` (or reuse from request-auth)
- Add private `function getUserAuth(options: GoogleAuthOptions): OAuth2Client | null`
  - If `options.accessToken`: create `new OAuth2Client()`, call `setCredentials({ access_token: options.accessToken })`, return it
  - Otherwise return null
- Modify `getSlidesClient(options?: GoogleAuthOptions)`:
  - If options provided: try `getUserAuth(options)`, if non-null use it as auth
  - Otherwise: use existing `getGoogleAuth()` (service account)
  - Return `google.slides({ version: 'v1', auth: userAuth ?? getGoogleAuth() })`
- Same pattern for `getDriveClient(options?: GoogleAuthOptions)` and `getDocsClient(options?: GoogleAuthOptions)`
- `verifyGoogleAuth()` remains unchanged (always uses service account)
- CRITICAL: The optional parameter means all 14+ existing call sites with no arguments continue working unchanged

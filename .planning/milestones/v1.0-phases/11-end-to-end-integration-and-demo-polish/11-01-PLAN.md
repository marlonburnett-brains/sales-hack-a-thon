**Critical pattern for all forms:** Use a monotonic Set for completed steps. Once a step ID is added to the completed set, never remove it. This prevents stepper flicker during polling.

**All forms need these new imports:**
```typescript
import { toast } from "sonner";
import { PipelineStepper } from "./pipeline-stepper"; // or "../touch/pipeline-stepper" for pre-call
import { TOUCH_N_PIPELINE_STEPS } from "./pipeline-steps"; // appropriate constant
import { mapToFriendlyError } from "@/lib/error-messages";
```

**All forms need these new state variables:**
```typescript
const [completedSteps, setCompletedSteps] = useState<Set<string>>(new Set());
const [activeStep, setActiveStep] = useState<string | null>(null);
const [errorStep, setErrorStep] = useState<string | null>(null);
const [errorMessage, setErrorMessage] = useState<string | null>(null);
```

**All forms: Update polling loops to derive step progress.** Inside each poll iteration, after getting `status`, add:
```typescript
const steps = status.steps ?? {};
// Monotonic: merge new completed steps into existing set
const newCompleted = new Set(completedSteps);
Object.entries(steps).forEach(([id, step]) => {
  if ((step as any).status === "completed") newCompleted.add(id);
});
setCompletedSteps(newCompleted);

// Find active step: first step in pipeline that is "running" or "waiting" and not completed
const pipelineSteps = TOUCH_N_PIPELINE_STEPS; // appropriate constant
const active = pipelineSteps.find(
  s => !newCompleted.has(s.id) && steps[s.id] && ((steps[s.id] as any).status === "running" || (steps[s.id] as any).status === "waiting")
);
setActiveStep(active?.id ?? null);
```

**All forms: Update error handling in catch blocks:**
```typescript
catch (err) {
  const raw = err instanceof Error ? err.message : "Generation failed";
  const friendly = mapToFriendlyError(raw);
  toast.error(friendly);
  setErrorStep(activeStep);
  setErrorMessage(friendly);
  setState("error-state-for-this-form"); // keep stepper visible
}
```

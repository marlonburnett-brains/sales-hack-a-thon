**3. Create `pollAssetPipeline` function:**
Add a new async function (above the render states section) that polls for asset pipeline progress. Pattern mirrors existing `pollStatus` but is dedicated to asset steps:
- Poll interval: 3000ms (3 seconds per locked decision)
- Max attempts: 120 (6 minutes)
- On each poll, call `checkTouch4StatusAction(currentRunId)`
- Use Monotonic Set pattern: `setCompletedSteps(prev => { const next = new Set(prev); Object.entries(steps).forEach(...); return next; })` -- use the functional updater form to avoid stale closure issues
- Find active step from `TOUCH_4_ASSET_PIPELINE_STEPS` array (not the extract/brief steps)
- Detect completion: `status.status === "suspended"` AND `steps["await-asset-review"]` exists with a payload -- this means all 7 asset steps finished and workflow suspended at HITL-2
- Also handle `status.status === "completed"` as completion (edge case)
- On completion: set `setActiveStep(null)`, wait 1500ms (all-green stepper display per locked decision), then `setState("awaitingAssetReview")` and call `router.refresh()`
- On failure (`status.status === "failed"`): show error on the last active step via `setErrorStep` and `setErrorMessage(mapToFriendlyError(...))`, show toast via `toast.error()`
- On timeout: `toast.error("Asset generation timed out")`

**4. Create `handleRetryAssetPipeline` function:**
Simple retry that re-polls (the workflow may have recovered server-side). Reset error state, transition to "assetGenerating", and call `pollAssetPipeline(runId)` again. If `runId` is null, show error toast.

**5. Replace `assetGenerating` render state (line 864-872):**
Replace the existing GenerationProgress placeholder with:
- A "Brief Approved" header: green CheckCircle icon + "Brief Approved" text (text-xs font-medium, flex items-center gap-1.5, text-green-600)

- `PipelineStepper` with `steps={TOUCH_4_ASSET_PIPELINE_STEPS}`, `completedStepIds={completedSteps}`, `activeStepId={activeStep}`, `errorStepId={errorStep}`, `errorMessage={errorMessage}`
- If `errorStep` is truthy, show a retry button: `<Button onClick={handleRetryAssetPipeline} variant="outline" className="w-full cursor-pointer gap-2"><RotateCcw className="h-4 w-4" />Try Again</Button>`

**6. Replace `awaitingAssetReview` render state (line 874-898):**
Replace the existing generic info alert with:
- Same "Brief Approved" green header as assetGenerating state
- A green success Alert (border-green-200 bg-green-50) with CheckCircle icon, title "Assets ready for review", description "All proposal assets have been generated. Review them before delivery."

The "approved" state should still exist as a fallback (e.g., if the workflow was already approved before the form loaded), but it should not tell users to go elsewhere -- the normal flow will never land here since handleApprove now transitions to assetGenerating.

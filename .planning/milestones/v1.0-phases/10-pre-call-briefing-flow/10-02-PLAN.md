    **Component 1: PreCallForm (`pre-call-form.tsx`)**
    A "use client" component following the three-state pattern (input -> loading -> result) from Touch 1 form.

    Props:
    ```typescript
    interface PreCallFormProps {
      dealId: string;
      companyName: string;
      industry: string;
    }
    ```

    State machine:
    - `idle`: Form visible with buyer role dropdown and meeting context textarea
    - `generating`: Loading spinner with "Preparing briefing..." message
    - `complete`: PreCallResults displayed with the generated briefing data
    - `error`: Error message with retry button

    Form fields (per locked decision):
    - **Company name**: Read-only display (pre-filled from deal, not editable)
    - **Industry**: Read-only display (pre-filled from deal)
    - **Buyer role**: shadcn/ui Select dropdown populated from BUYER_PERSONAS constant (import from @lumenalta/schemas). Default to "General".
    - **Meeting context**: shadcn/ui Textarea with placeholder "Describe the meeting agenda, goals, previous conversations, or any concerns..."

    Submit flow:
    1. Call `generatePreCallBriefingAction(dealId, { companyName, industry, buyerRole, meetingContext })`
    2. Get back `{ runId }`, switch to `generating` state
    3. Poll `checkPreCallStatusAction(runId)` every 2 seconds (same pattern as touch-1-form.tsx pollStatus)
    4. Max 60 attempts (2 min timeout)
    5. On `completed`: Extract generatedContent from the workflow result. The workflow's final step outputs the full briefing data. Parse the result to extract companyResearch, hypotheses, discoveryQuestions, caseStudies, docUrl.
    6. Switch to `complete` state and render PreCallResults
    7. On `failed`: Switch to `error` state

    Use shadcn/ui components: Card, CardHeader, CardTitle, CardContent, Button, Select, SelectTrigger, SelectValue, SelectContent, SelectItem, Textarea, Label.
    Use lucide-react icons: Sparkles (submit button), Loader2 (loading), AlertCircle (error).

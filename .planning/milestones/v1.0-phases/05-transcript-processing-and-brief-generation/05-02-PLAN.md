3. Step 1 -- `parseTranscript`:
   - id: "parse-transcript"
   - Uses Gemini 2.5 Flash with TranscriptFieldsLlmSchema via zodToGeminiSchema
   - Prompt must instruct Gemini to:
     - Extract INDIRECT/implied mentions (not just explicit statements)
     - Use industry and subsector context to interpret domain-specific language
     - Treat additionalNotes as supplementary context
     - Return empty string ONLY when the topic is completely absent from the transcript
     - Include the full transcript text in the prompt
   - Outputs: all input fields pass-through + extractedFields (TranscriptFields)

4. Step 2 -- `validateFields`:
   - id: "validate-fields"
   - Pure logic step (no LLM call)
   - Computes fieldSeverity: `Record<string, "error" | "warning" | "ok">` using tiered severity per user decision:
     - customerContext and businessOutcomes: "error" if empty string after trim, "ok" otherwise
     - constraints, stakeholders, timeline, budget: "warning" if empty string after trim, "ok" otherwise
   - Passes through all fields + adds fieldSeverity
   - Also adds hasErrors boolean (true if any field has "error" severity)

5. Step 3 -- `awaitFieldReview`:
   - id: "await-field-review"
   - suspendSchema: z.object({ reason: z.string(), extractedFields: TranscriptFieldsLlmSchema, fieldSeverity: record of severity, hasErrors: z.boolean(), dealId: z.string() })
   - resumeSchema: z.object({ reviewedFields: TranscriptFieldsLlmSchema })
   - On first execution (no resumeData): suspend with extracted fields, severity map, and dealId
   - On resume: pass through all input data + reviewedFields from resumeData

6. Chain the 3 steps: `.then(parseTranscript).then(validateFields).then(awaitFieldReview)`
   - DO NOT call `.commit()` yet -- export the workflow builder so Plan 03 can add remaining steps and commit
   - Actually, Mastra requires `.commit()` at definition time. Instead: define all 3 steps as named exports, AND export a partial workflow with just these 3 steps + placeholder final steps. Plan 03 will replace the entire workflow file with all steps.
   - CORRECTION: Export the workflow with `.commit()` including these 3 steps. The workflow output after awaitFieldReview will be: { dealId, companyName, industry, subsector, transcript, additionalNotes, extractedFields, fieldSeverity, hasErrors, reviewedFields, decision: "continued" }. Plan 03 will add the remaining steps by appending to the .then() chain before .commit().

7. Register in `apps/agent/src/mastra/index.ts`:
   - Import touch4Workflow from "./workflows/touch-4-workflow"
   - Add `"touch-4-workflow": touch4Workflow` to the workflows config

NOTE: The workflow will only have the first 3 steps for now. After seller field review, the workflow will complete with the reviewedFields as output. Plan 03 extends this to add pillar mapping, brief generation, ROI framing, and interaction recording.

Add 3 new steps to the existing touch-4-workflow.ts (after the awaitFieldReview step from Plan 02) and update the workflow chain.

1. Step 4 -- `mapPillarsAndGenerateBrief` (combined Gemini call using SalesBriefLlmSchema):
   - id: "map-pillars-generate-brief"
   - Input: pass-through from awaitFieldReview including reviewedFields, companyName, industry, subsector
   - Uses Gemini 2.5 Flash with SalesBriefLlmSchema via zodToGeminiSchema
   - Prompt must:
     - Provide the 6 Lumenalta SOLUTION_PILLARS as context for pillar mapping
     - Use the seller-REVIEWED fields as ground truth (not raw transcript)
     - Include company name, industry, and subsector for contextual accuracy
     - Instruct Gemini to select a primary pillar and at least one secondary pillar with evidence
     - Generate 2-4 use cases specific to the customer's situation
     - Each use case needs name, description, initial roiOutcome, and valueHypothesis
     - Ground all content in the transcript data -- no hallucinated capabilities
   - Parse response with SalesBriefLlmSchema
   - Output: all pass-through fields + brief (SalesBrief type)

2. Step 5 -- `generateROIFraming` (enrichment Gemini call using ROIFramingLlmSchema):
   - id: "generate-roi-framing"
   - Input: pass-through from mapPillarsAndGenerateBrief including brief
   - Uses Gemini 2.5 Flash with ROIFramingLlmSchema via zodToGeminiSchema
   - Prompt must:
     - Take the use cases from the generated brief as input
     - Generate 2-3 SPECIFIC, quantifiable ROI outcome statements per use case
     - Generate a value hypothesis per use case connecting Lumenalta's delivery model to customer needs
     - ROI outcomes should reference industry benchmarks where applicable (e.g., "Reduce processing time by 40%")
   - Parse response with ROIFramingLlmSchema
   - Output: all pass-through fields + roiFraming (ROIFraming type)

3. Step 6 -- `recordInteraction` (database persistence, no LLM):
   - id: "record-interaction"
   - Creates all database records in a single step:
     a. InteractionRecord: touchType "touch_4", status "completed", decision "completed", inputs as JSON (companyName, industry, subsector), generatedContent as JSON summary (primaryPillar, secondaryPillars, useCaseCount), outputRefs as null (no Drive artifacts for briefs)
     b. Transcript: linked to InteractionRecord via interactionId, rawText from input transcript, additionalNotes, subsector, all 6 reviewed field values from reviewedFields
     c. Brief: linked to InteractionRecord via interactionId, primaryPillar, secondaryPillars as JSON string, evidence, all 6 reviewed field values, useCases as JSON string (from brief), roiFraming as JSON string (merged with brief use cases)
     d. FeedbackSignal: linked to InteractionRecord, signalType "positive", source "touch_4_complete", content as JSON with pillar mapping summary
   - Output: { interactionId, transcriptId, briefId, briefData (the full brief object for UI display), roiFramingData (the full ROI framing for UI display) }

4. Update the workflow chain:
   ```typescript
   export const touch4Workflow = createWorkflow({...})
     .then(parseTranscript)
     .then(validateFields)
     .then(awaitFieldReview)
     .then(mapPillarsAndGenerateBrief)

     .then(recordInteraction)
     .commit();
   ```


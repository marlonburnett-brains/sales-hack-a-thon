     ```
       industry: string;
       subsector: string;
       primaryPillar: string;
       secondaryPillars: string[];
       useCases: { name: string; description: string }[];
       limit?: number;
     }): Promise<ProposalSearchResult>
     ```

     ```
     { candidates: SlideSearchResult[]; primaryCount: number; secondaryCount: number; caseStudyCount: number; }
     ```
   - Implementation: THREE retrieval passes using existing `searchSlides()`:
     - **Pass 1 (Primary pillar):** Query = `"${primaryPillar} ${industry} solution proposal capabilities"`, industry filter = industry, limit = limit ?? 20. Store results in a Map keyed by slideId for deduplication.
     - **Pass 2 (Secondary pillars):** For each secondary pillar, query = `"${pillar} ${industry}"`, industry = industry, limit = 5. Add to Map (dedup by slideId).
     - **Pass 3 (Case studies):** Query = `"case study ${industry} ${subsector} results outcomes"`, industry = industry, limit = 5. Add to Map.
   - Return: `{ candidates: Array.from(map.values()), primaryCount, secondaryCount, caseStudyCount }`
   - Three-tier fallback for sparse results (per CONTEXT.md discretion):
     - If primary returns < 3 slides, re-query WITHOUT industry filter (broader search)
     - If still < 3 total, query `"${primaryPillar} capabilities solutions"` (cross-industry)
     - Never fail -- always return whatever candidates are found, even if sparse
   - Export `ProposalSearchResult` type

2. **Create `apps/agent/src/lib/proposal-assembly.ts`** -- business logic module:

   a. **filterByMetadata()**: Post-retrieval metadata filtering function
      - Signature: `filterByMetadata(candidates: SlideSearchResult[], industry: string, pillar: string): SlideSearchResult[]`
      - Parse each candidate's `metadata` field using `SlideMetadataSchema.safeParse()`
      - Keep slides where: industry tag includes the brief's industry OR industries array is empty (cross-industry), AND solutionPillars array includes the pillar (case-insensitive includes match)

   b. **buildSlideJSON()**: Assemble the SlideJSON intermediate representation
      - Signature: `buildSlideJSON(params: { brief: SalesBrief; roiFraming: ROIFraming; selectedSlides: SlideSearchResult[]; }): SlideAssembly`
      - Fixed section template order (per locked decision):
        1. title_context (1-2 synthesized slides from brief data)
        2. problem_restatement (1 synthesized slide)
        3. primary_capability (slides matching primary pillar)
        4. secondary_capability (slides matching secondary pillars)
        5. case_study (slides with content_type = "case_study" metadata)
        6. roi_outcomes (1 synthesized slide from ROI framing data)
        7. next_steps (1 synthesized slide)
      - Synthesized slides: sourceBlockRef = "" (empty string), sourceType = "synthesized"
      - Retrieved slides: sourceBlockRef = slide.slideId, sourceType = "retrieved"
      - Title slide: `slideTitle = "${brief.companyName} - Solution Proposal"`, bullets = [industry, primary pillar, customerContext excerpt (first 200 chars)], speakerNotes = "Opening context slide. Restate the customer's situation before presenting solutions."
      - Problem restatement slide: `slideTitle = "Understanding Your Challenges"`, bullets from brief.businessOutcomes and brief.constraints, speakerNotes from brief.customerContext
      - ROI outcomes slide: `slideTitle = "Expected Business Impact"`, bullets from roiFraming.useCases[].roiOutcomes (flatten), speakerNotes summarizing value hypotheses
      - Next steps slide: `slideTitle = "Recommended Next Steps"`, bullets = ["Discovery deep-dive on priority use cases", "Technical architecture workshop", "Preliminary timeline and resource planning"], speakerNotes = "Close with clear next actions."
      - Dynamic deck length formula (per discretion): `2 (synthesized context) + 1 (problem) + min(3, useCases.length) * 2 + min(2, caseStudySlides.length) + 1 (ROI) + 1 (next steps)`, floor 8 ceiling 18
      - For retrieved slides: preserve original slideTitle, use textContent as bullets (split by newline), use speakerNotes from search result

   c. **generateSlideCopy()**: Per-slide copy generation via Gemini
      - Signature: `generateSlideCopy(params: { slide: { slideTitle: string; textContent: string; speakerNotes: string; sourceBlockRef: string; }; brief: SalesBrief; brandGuidelines: string; }): Promise<ProposalCopy>`
      - Uses GoogleGenAI SDK with Gemini 2.5 Flash
      - Prompt structure:
        - System context: "You are a copywriter at Lumenalta generating slide content for a custom solution proposal."
        - Brand voice guidelines hardcoded in prompt (per discretion): "Lumenalta brand voice: Professional, outcome-focused, concise. Avoid jargon. Lead with business outcomes. Use active voice. Be specific with metrics and results."
        - Source slide content (the original text from AtlusAI, truncated to 500 chars)
        - Brief context (companyName, industry, customerContext, businessOutcomes -- the customer's actual situation)
        - Explicit constraint: "ONLY use information from the provided source content and brief. Do NOT introduce new capabilities, statistics, or claims not present in either source."
        - Instructions: "1. Preserve the slide title (lightly edit only if needed for customer context). 2. Rewrite bullet text to connect the source capabilities to this customer's specific needs. 3. Generate fresh speaker notes with talking points."
      - Uses `zodToGeminiSchema(ProposalCopyLlmSchema)` for structured output
      - Parses response with `ProposalCopyLlmSchema.parse()`
      - If source slide has fewer than 20 words of textContent, skip copy rewriting and return original text as-is (per discretion for minimal content slides)
      - Import from `@lumenalta/schemas`: ProposalCopyLlmSchema, zodToGeminiSchema, SalesBriefLlmSchema types
      - Import env from `../../env` (NOT `../env` -- proposal-assembly.ts is in lib/)

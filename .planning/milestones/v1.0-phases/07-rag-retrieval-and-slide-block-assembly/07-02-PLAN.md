**Step 9: ragRetrieval**
- id: "rag-retrieval"
- inputSchema: matches finalizeApproval outputSchema (interactionId, transcriptId, briefId, briefData: SalesBriefLlmSchema, roiFramingData: ROIFramingLlmSchema, decision: z.enum(["approved"]), reviewerName: z.string())
- outputSchema: z.object({
    interactionId: z.string(),
    briefId: z.string(),
    briefData: SalesBriefLlmSchema,
    roiFramingData: ROIFramingLlmSchema,
    candidateSlides: z.string(), // JSON-serialized SlideSearchResult[] (too complex for Zod workflow schema, store as JSON string)
    retrievalSummary: z.string(),
  })
- execute:
  1. Fetch Brief from DB: `prisma.brief.findUniqueOrThrow({ where: { id: inputData.briefId }, include: { interaction: { include: { deal: { include: { company: true } } } } } })`
  2. Get industry from `brief.interaction.deal.company.industry`
  3. Get subsector: parse `brief.interaction.transcript` relation or fallback to brief's subsector if available. Since Transcript has a `subsector` field, fetch it: `prisma.transcript.findFirst({ where: { interactionId: brief.interactionId } })`
  4. Call `searchForProposal({ industry, subsector: transcript?.subsector ?? "", primaryPillar: inputData.briefData.primaryPillar, secondaryPillars: inputData.briefData.secondaryPillars, useCases: inputData.briefData.useCases })`
  5. Call `filterByMetadata(result.candidates, industry, inputData.briefData.primaryPillar)` to narrow candidates
  6. Build retrievalSummary string: "Retrieved {total} candidates ({primary} primary, {secondary} secondary, {caseStudy} case studies). Post-filter: {filtered} slides."
  7. Serialize filtered candidates as JSON string for workflow passthrough
  8. Return output with candidateSlides as JSON string, briefData/roiFramingData passed through
- Import: searchForProposal from `../../lib/atlusai-search`, filterByMetadata from `../../lib/proposal-assembly`

  5. Return output with slideJSON, slideCount, briefData/roiFramingData passed through

**Step 11: generateCustomCopy**
- id: "generate-custom-copy"
- inputSchema: matches assembleSlideJSON outputSchema
- outputSchema: z.object({
    interactionId: z.string(),
    briefId: z.string(),
    slideJSON: z.string(), // JSON-serialized SlideAssembly with bespoke copy applied
    slideCount: z.number(),
    retrievalSummary: z.string(),
  })
- execute:
  1. Deserialize slideJSON from JSON string to SlideAssembly
  2. For each slide in the array:
     - If sourceType === "synthesized": skip copy generation (synthesized slides already have final content from buildSlideJSON)
     - If sourceType === "retrieved": call `generateSlideCopy({ slide: { slideTitle: s.slideTitle, textContent: s.bullets.join("\n"), speakerNotes: s.speakerNotes, sourceBlockRef: s.sourceBlockRef }, brief: inputData.briefData, brandGuidelines: BRAND_GUIDELINES })`
     - Replace the slide's slideTitle, bullets, and speakerNotes with the generated copy
  3. Define `BRAND_GUIDELINES` as a constant string at the top of the step (not in a separate file -- per discretion, hardcoded in prompt is simplest):
     ```
     "Lumenalta brand voice: Professional, outcome-focused, concise. Avoid jargon and buzzwords. Lead with business outcomes, not technology features. Use active voice. Be specific with metrics and results when available. Address the customer's situation directly."
     ```
  4. Process slides sequentially (per-slide, not batch -- per research discretion for quality). Use a for...of loop, not Promise.all (avoids rate limiting and maintains quality).
  5. Re-serialize the updated SlideAssembly to JSON string
  6. Return output with updated slideJSON, slideCount, retrievalSummary passed through
  7. Log progress: "Generating copy for slide {i+1}/{total}: {slideTitle}"
- Import: generateSlideCopy from `../../lib/proposal-assembly`

Calls selectSlidesForDeck with touchType='touch_2'. Returns IntroDeckSelection (selectedSlideIds, slideOrder, personalizationNotes).
       - **assembleDeck**: Calls assembleDeckFromSlides with:
         - sourcePresentationId: env.MEET_LUMENALTA_PRESENTATION_ID (the Meet Lumenalta deck to select from -- add this env var)
         - selectedSlideIds and slideOrder from previous step
         - targetFolderId: obtained via getOrCreateDealFolder
         - deckName: `${companyName} - Meet Lumenalta - ${new Date().toISOString().split('T')[0]}`
         - customizations: { salespersonName, salespersonPhotoUrl, customerName, customerLogoUrl }
       - **recordInteraction**: Creates InteractionRecord in Prisma with touchType='touch_2', status='approved' (Touch 2/3 are direct generation per locked decision -- no intermediate review step), inputs (all form data as JSON), outputRefs (driveUrl), driveFileId (presentationId). Creates FeedbackSignal with signalType='positive'. Ingests deck via ingestGeneratedDeck.
       - Returns { presentationId, driveUrl, interactionId }

    2. Use createWorkflow + createStep pattern from Mastra. Input schema: z.object({ dealId, companyName, industry, salespersonName, salespersonPhotoUrl?, customerName?, customerLogoUrl?, context?, priorTouchOutputs? }). Output schema includes presentationId, driveUrl.

    **Touch 3 workflow (apps/agent/src/mastra/workflows/touch-3-workflow.ts):**

    3. Same pattern as Touch 2 with these differences:
       - **selectSlides**: Calls selectSlidesForDeck with touchType='touch_3' and capabilityAreas param
       - **assembleDeck**: Uses a capability deck source presentation (env.CAPABILITY_DECK_PRESENTATION_ID or falls back to the same template). Deck name: `${companyName} - Capability Alignment - ${capabilityAreas.join(', ')} - ${date}`
       - **recordInteraction**: touchType='touch_3', includes capabilityAreas in inputs
       - Input schema includes capabilityAreas: z.array(z.string())

    **Register workflows:**

    4. In apps/agent/src/mastra/index.ts, import touch2Workflow and touch3Workflow and add to the workflows object alongside touch-1-workflow.

    5. Add MEET_LUMENALTA_PRESENTATION_ID and CAPABILITY_DECK_PRESENTATION_ID (optional, defaults to template) to apps/agent/src/env.ts. These are the source presentations that Touch 2/3 select slides from.

    **Key design decision per locked decisions:**
    - Touch 2/3 are direct end-to-end generation: AI selects slides and assembles the final deck WITHOUT an intermediate slide review step (unlike Touch 1 which has a two-step review)
    - Seller reviews the final generated deck via embedded iframe preview on the deal page
    - Revision path: seller can regenerate with tweaked inputs, creating a NEW version (old versions remain in Drive)
    - Each regeneration creates a new InteractionRecord

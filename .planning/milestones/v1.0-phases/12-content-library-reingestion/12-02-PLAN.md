    This reads the existing `content-manifest.json` (populated by Plan 12-01's manifest-only run) and creates Google Docs in the `_slide-level-ingestion` Drive folder for each "pending" slide. AtlusAI auto-indexes these documents.

    Monitor the output for:
    - Total slides ingested vs errors
    - Any rate limit (429) or permission (403) errors
    - Final manifest summary

    ```bash
    cd apps/agent && npx tsx --env-file=.env src/ingestion/ingest-brand-guidelines.ts
    ```

    If the Google Docs API is properly enabled (from Plan 12-01 Task 1), this should succeed. If it still fails with 403, log the error and note it as a known issue.

    **Step 3: Run image registry builder.** From `apps/agent`:
    ```bash
    cd apps/agent && npx tsx --env-file=.env src/ingestion/build-image-registry.ts
    ```
    This discovers curated image folders (headshots, logos, icons) in the accessible Drive scope and upserts them into the Prisma ImageAsset table. Per user decision: if curated image folders still are not accessible, accept an empty registry -- do NOT attempt to catalog the ~9,000 files in `01 Resources/`.

    This will re-classify and regenerate `coverage-report.json` without re-ingesting (already-ingested slides keep their status).

    **Step 5: Log the final state.** Print a summary of:
    - Total slides ingested (from manifest: count where status = "ingested")
    - Total slides errored (from manifest: count where status = "error")
    - Total presentations
    - Case study count per industry (from coverage report)
    - Template count per industry
    - Brand guidelines ingestion status (success/failure)
    - Image registry asset count (from Prisma or image-registry-report.json)
    - Industries with gaps (if any)

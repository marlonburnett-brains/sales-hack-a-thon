  <action>
    Add the `ImageAsset` model to `apps/agent/prisma/schema.prisma` after the existing `WorkflowJob` model:

    ```prisma
    model ImageAsset {
      id          String   @id @default(cuid())
      category    String   // "headshot" | "logo" | "icon" | "brand_element"
      name        String   // Person name or asset name (e.g., "John Smith", "Lumenalta Logo Primary")
      description String?  // Optional description for search
      driveFileId String   @unique // Google Drive file ID — prevents duplicate entries
      driveUrl    String   // Direct Google Drive URL: https://drive.google.com/file/d/{id}/view
      mimeType    String   // "image/png", "image/jpeg", etc.
      tags        String   // JSON array of tags: ["leadership", "executive"]
      createdAt   DateTime @default(now())
      updatedAt   DateTime @updatedAt

      @@index([category])
      @@index([name])
    }
    ```

    Then run the Prisma migration:
    ```bash
    cd apps/agent && npx prisma migrate dev --name add-image-asset
    ```

    Then regenerate the Prisma client:
    ```bash
    cd apps/agent && npx prisma generate
    ```

  <action>
    **Step 1 — Image registry build script:**
    Create `apps/agent/src/ingestion/build-image-registry.ts` — runnable via `npx tsx apps/agent/src/ingestion/build-image-registry.ts`.

    The script discovers and catalogs curated image assets from the Google Drive folder:

    1. Use `getDriveClient()` to list files in the Drive folder, looking for image-related subfolders. The CONTEXT.md mentions `01 Resources/` contains ~9,000 files — do NOT ingest all of them. Focus on:
       - Folders with names containing "headshot", "team", "leadership", "photo", "logo", "brand", "icon"
       - Files with image mimeTypes: image/png, image/jpeg, image/svg+xml, image/gif
    2. Apply curation rules (per locked decision — curated subset only):
       - Leadership headshots: files in folders matching "headshot" or "team" or "leadership" patterns
       - Company logos: files in folders matching "logo" or "brand" patterns
       - Brand icons: files in folders matching "icon" patterns
       - Skip files that appear to be duplicates (same name with size variants like "John Smith (1).jpg")
       - Skip stock photos (files not matching any curated category)
    3. For each curated file, determine:
       - `category`: "headshot", "logo", "icon", or "brand_element" based on folder name
       - `name`: cleaned file name (remove extension, clean up formatting)
       - `driveUrl`: construct from file ID: `https://drive.google.com/file/d/${fileId}/view`
       - `tags`: derive from folder path (e.g., file in "Leadership/Headshots" -> ["leadership", "headshot"])
    4. Upsert into the ImageAsset Prisma table using `driveFileId` as the unique key (idempotent):
       ```typescript
       await prisma.imageAsset.upsert({
         where: { driveFileId: file.id },
         create: { ... },
         update: { ...updated fields... },
       })
       ```
    5. Handle Drive API pagination (nextPageToken) for each folder
    6. Use `supportsAllDrives: true` and `includeItemsFromAllDrives: true` on all Drive API calls
    7. Log summary: total files discovered, total curated, total by category, total skipped
    8. Write a JSON report to `apps/agent/src/ingestion/manifest/image-registry-report.json`:
       ```json
       {
         "generatedAt": "ISO timestamp",
         "totalDiscovered": N,
         "totalCurated": N,
         "totalSkipped": N,
         "byCategory": { "headshot": N, "logo": N, "icon": N, "brand_element": N },
         "items": [{ "name": "...", "category": "...", "driveUrl": "..." }, ...]
       }
       ```

    **Step 2 — Brand guidelines ingestion:**
    Create `apps/agent/src/ingestion/ingest-brand-guidelines.ts` — runnable via `npx tsx apps/agent/src/ingestion/ingest-brand-guidelines.ts`.

    This script ingests the "Branded Basics" document (brand guidelines) into AtlusAI as a whole reference document (per Claude's Discretion recommendation from RESEARCH.md — keep as whole reference, do NOT extract structured rules):

    1. Find the "Branded Basics" presentation in the Drive folder by name (search for files matching "Branded Basics" or "Brand Guide")
    2. If it's a Google Slides presentation: extract ALL slides' text content as one concatenated document
    3. If it's a Google Doc or PDF: note this and attempt to read content via appropriate API
    4. Create a single AtlusAI document with:
       - documentId: deterministic hash of "brand-guidelines:{presentationId}"
       - Full text content of the entire document
       - Metadata: `{ contentType: "brand_guide", industries: ["all"], touchType: ["touch_1", "touch_2", "touch_3", "touch_4"] }`
    5. Use `ingestDocument()` from `atlusai-client.ts` — if Plan 02-01 has not completed yet, wrap the AtlusAI ingestion call in a try/catch and log a warning: "AtlusAI client not available yet — run this script again after Plan 02-01 completes"

    NOTE: The brand guidelines ingestion depends on `atlusai-client.ts` from Plan 02-01. Since this plan is Wave 1 (parallel with 02-01), the AtlusAI ingestion step may need to be re-run after 02-01 completes. The Prisma model and image registry build are fully independent and will complete regardless.

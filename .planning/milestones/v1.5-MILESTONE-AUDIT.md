### Critical Integration Path: Classification -> Deck Inference

The most important cross-phase connection is fully wired:
1. Phase 33 `POST /templates/:id/classify` writes `contentClassification` and `touchTypes` to Template model
2. Phase 34 `inferDeckStructure()` queries by `contentClassification: "example"` and filters by `touchTypes`
3. Phase 34 loads Phase 33 data products: `SlideEmbedding.description` and `SlideElement` relations
4. Phase 34 cron detects classification changes via SHA-256 hash and triggers re-inference

### Shared Component Wiring

- Phase 32 `IngestionStatusBadge` consumed by both Discovery and Templates pages
- Phase 33 adds "classify" status to shared `STATUS_CONFIG` -- renders through existing badge component

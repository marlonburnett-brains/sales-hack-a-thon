**Create the route and main viewer components.**

**page.tsx** — Server Component entry point:
- Accept `params: Promise<{ id: string }>` (Next.js 15 async params pattern)
- Call `listSlidesAction(id)` and `getSlideThumbnailsAction(id)` in parallel using `Promise.all`
- Also fetch template name: call the existing `listTemplatesAction()` and find by id, or add a simple lookup. Use the template name for the page heading.
- Render `<SlideViewerClient templateId={id} templateName={name} initialSlides={slides} initialThumbnails={thumbnails} />`
- If no slides found, show an empty state message: "No ingested slides. Trigger ingestion from the Templates page."

**slide-viewer-client.tsx** — "use client" main viewer:
- State: `currentIndex` (number), `slides` (SlideData[]), `thumbnails` (Map<string, string> keyed by slideObjectId)
- Layout per user decisions:
  - Slide preview area (~70% width) + right sidebar panel (~30% width)
  - Horizontal thumbnail strip at bottom spanning full width
  - Slide counter "3 of 12" in the top area
- Navigation:
  - Left/right arrow buttons (ChevronLeft, ChevronRight from lucide-react) flanking the slide
  - Keyboard listener: ArrowLeft, ArrowRight for navigation, T for thumbs up, R for reject (Claude's discretion shortcuts)
  - Clicking a thumbnail in the strip jumps to that slide
- Pass current slide data + thumbnail URL to `<SlidePreview>`
- Pass current slide data to `<ClassificationPanel>` (built in Task 2)
- Pass thumbnails array + currentIndex + onSelect callback to `<ThumbnailStrip>`
- Template name as breadcrumb/heading at the top

**slide-preview.tsx** — Slide image display:
- Props: `{ thumbnailUrl: string | undefined; slideIndex: number; isLoading: boolean }`
- Render the slide thumbnail as an `<img>` at max width, maintaining 16:9 aspect ratio
- Use `aspect-video` Tailwind class for the container
- Show `<Skeleton>` while thumbnail loads or if URL is undefined
- The image should be rendered at presentation size (up to 1600px wide from Google Slides API LARGE thumbnail)

**thumbnail-strip.tsx** — Horizontal thumbnail navigation:
- Props: `{ thumbnails: Array<{ slideIndex: number; thumbnailUrl: string }>; currentIndex: number; onSelect: (index: number) => void }`
- Horizontal scrollable strip with overflow-x-auto
- Each thumbnail rendered as a small image (~120px wide, maintaining aspect ratio)
- Active slide highlighted with a blue border ring (ring-2 ring-blue-500)
- Other thumbnails have a subtle border (border border-slate-200)
- Clicking a thumbnail calls `onSelect(slideIndex)`
- Auto-scroll to keep active thumbnail visible using `scrollIntoView({ behavior: "smooth", block: "nearest" })`
- Use flex-shrink-0 for each thumbnail to prevent compression

**classification-panel.tsx** — Right sidebar panel:

- Display classification tags grouped by category with colored chips:
  - Industry: blue chips (bg-blue-50 text-blue-700 border-blue-200)
  - Solution Pillar: purple chips (bg-purple-50 text-purple-700)
  - Buyer Persona: green chips (bg-green-50 text-green-700)
  - Funnel Stage: amber chips (bg-amber-50 text-amber-700)
  - Content Type: slate chips (bg-slate-100 text-slate-700)
  - Slide Category: indigo chips (bg-indigo-50 text-indigo-700)
- Each category has a label header (text-xs font-medium text-slate-500 uppercase tracking-wider)
- Confidence score displayed as "85% confident" with a small Progress bar (shadcn) underneath
- Review status indicator: show current status as a small badge
- Rating buttons:
  - ThumbsUp and ThumbsDown icons from lucide-react
  - Side-by-side buttons at the top of the panel
  - Active state: filled/colored when selected (ThumbsUp green, ThumbsDown red)
  - Thumbs up: call `updateSlideClassificationAction(slide.id, templateId, { reviewStatus: "approved" })`, show sonner toast "Classification approved", call `onUpdated()`
  - Thumbs down: switch to editing mode (show TagEditor), do NOT save yet — user edits tags first, then saves
- When in editing mode (thumbs down clicked):
  - Show `<TagEditor>` replacing the static chips
  - "Save Corrections" button at the bottom
  - "Cancel" button to exit edit mode without saving

**tag-editor.tsx** — Inline tag correction:
- Props: `{ currentTags: CorrectedTags; onSave: (tags: CorrectedTags) => void; onCancel: () => void; isSaving: boolean }`
- Use shadcn Select component for each category
- Multi-value categories (industries, solutionPillars, buyerPersonas, funnelStages): Use multiple Select dropdowns or a custom multi-select pattern. Since shadcn Select is single-value by default, implement a "click chip to remove + Select to add" hybrid approach:
  - Show existing values as removable chips (click X to remove)
  - Show a Select dropdown below to add a new value (filtered to exclude already-selected)
  - Populate options from `@lumenalta/schemas` constants (INDUSTRIES, SOLUTION_PILLARS, BUYER_PERSONAS, FUNNEL_STAGES)
- Single-value categories (contentType, slideCategory): Use standard shadcn Select
  - Populate from CONTENT_TYPES and SLIDE_CATEGORIES constants
- Import constants from `@lumenalta/schemas` (the workspace package)
- On "Save Corrections": call `onSave(editedTags)` which the parent ClassificationPanel wires to `updateSlideClassificationAction` with `reviewStatus: "needs_correction"`
- Show loading state on save button while `isSaving` is true

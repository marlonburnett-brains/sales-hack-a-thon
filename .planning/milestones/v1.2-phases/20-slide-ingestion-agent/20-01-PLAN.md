**1. embed-slide.ts (NEW):** Create Vertex AI embedding generation function.
- Export `generateEmbedding(text: string): Promise<number[]>`
- Use `@google/genai` with `vertexai: true`, project from `env.GOOGLE_CLOUD_PROJECT`, location from `env.GOOGLE_CLOUD_LOCATION`
- Model: `text-embedding-005`, taskType: `RETRIEVAL_DOCUMENT`
- Returns `number[768]` from `response.embeddings![0]!.values!`
- Handle empty/very short text: if input is empty string, return a zero vector of 768 dimensions (don't call API)

**2. smart-merge.ts (NEW):** Content hash computation and merge logic.
- Export `computeContentHash(textContent: string, speakerNotes: string, slideObjectId: string): string` -- SHA-256 of concatenated content, truncated to 40 chars. Include slideObjectId to prevent collisions on empty/low-content slides (per research pitfall 4).
- Export `computeMerge(newSlides: ExtractedSlide[], existingEmbeddings: ExistingSlideData[]): MergeResult`
- `MergeResult` interface: `{ unchanged: {existing, newIndex}[], changed: {existing, slide}[], added: ExtractedSlide[], toArchive: ExistingSlideData[] }`
- `ExistingSlideData` interface: `{ id: string, contentHash: string | null, slideIndex: number, confidence: number | null, classificationJson: string | null }`
- Match by contentHash. Slides in new but not existing = added. Slides in existing but not new = toArchive. Same hash different index = unchanged (update index). Same hash same index = unchanged (no-op).

**3. ingest-template.ts (NEW):** Main orchestrator function.
- Export `ingestTemplate(templateId: string): Promise<IngestResult>` where `IngestResult = { processed: number, skipped: number, archived: number }`
- Steps:
  a. Fetch template from DB (`prisma.template.findUniqueOrThrow`)
  b. Set ingestionStatus to "ingesting" and ingestionProgress to `{ phase: "extracting", current: 0, total: 0 }`
  c. Call `extractSlidesFromPresentation(template.presentationId, template.name, "")`
  d. Update progress: `{ phase: "classifying", current: 0, total: slides.length }`
  e. Fetch existing SlideEmbeddings for this templateId (id, contentHash, slideIndex, confidence, classificationJson)
  f. Call `computeMerge(slides, existingEmbeddings)`
  g. For unchanged slides: just update slideIndex if changed via raw SQL UPDATE
  h. For added + changed slides: loop sequentially with 300ms delay between iterations:
     - Get title slide text (slides[0].textContent) for classification context
     - Call `classifySlide(slide, titleSlideText, [])` -- pass empty pillar list (pillars come from the presentation context)
     - Add confidence to classification: modify the classifySlide call or add confidence to the GEMINI_RESPONSE_SCHEMA (see Task 2 note on classify-metadata.ts)
     - Call `generateEmbedding(slide.textContent + " " + slide.speakerNotes)`
     - Upsert into SlideEmbedding using `prisma.$executeRaw` with `pgvector.toSql()` for the embedding column (per research: Prisma ORM cannot handle vector type)
     - For changed slides: set needsReReview=true, store previous classificationJson in previousClassificationJson, lower confidence by 50%
     - Update progress after each slide: `{ phase: "classifying", current: N, total: M, skipped: S }`
  i. Archive removed slides: UPDATE SET archived=true
  j. Clear progress, set ingestionStatus="idle", update lastIngestedAt, slideCount
  k. Wrap entire function in try/catch: on error, set ingestionStatus="failed", clear progress, log error

**4. ingestion-queue.ts (NEW):** Simple in-memory sequential queue.
- Export `ingestionQueue` singleton instance of `IngestionQueue` class
- Methods: `enqueue(templateId: string): void`, `isProcessing(): boolean`
- Deduplicates: if templateId already in queue, skip
- Processes one at a time: calls `ingestTemplate()`, then processes next
- On error: logs and continues to next item
- On startup cleanup: export `clearStaleIngestions(): Promise<void>` that finds templates with ingestionStatus != "idle" and resets them (handles crash recovery per research pitfall 5)

**5. classify-metadata.ts (MODIFY):** Add confidence score to Gemini response schema.
- Add `confidence` field to `GEMINI_RESPONSE_SCHEMA`:
  ```
  confidence: {
    type: Type.NUMBER,
    description: "Overall confidence score (0-100) for this classification. 100 = highly confident all tags are correct, 50 = moderate confidence, below 30 = low confidence/ambiguous content."
  }
  ```
- Add "confidence" to the `required` array
- Update `ClassifiedSlide` interface to include `confidence: number` (or add it to the return)
- In `classifySlide()`: extract confidence from parsed response, clamp to 0-100, include in return value
- Keep the existing default metadata fallback but add `confidence: 50` as default on parse failure

- Add state: `const [progress, setProgress] = useState<{current: number, total: number} | null>(null)`
- Add useEffect that polls when status is "ingesting":
  ```
  useEffect(() => {
    if (status !== "ingesting") { setProgress(null); return; }
    const interval = setInterval(async () => {
      const p = await getIngestionProgressAction(template.id);
      if (p && p.current > 0) setProgress({ current: p.current, total: p.total });
      if (!p || p.status === "idle" || p.status === "failed") {
        clearInterval(interval);
        setProgress(null);
        onRefresh?.();
        if (p?.status === "idle") {
          toast.success(`${template.name} ingested (${p.total} slides)`);
        }
      }
    }, 2000);
    return () => clearInterval(interval);
  }, [status]);
  ```
- Add `onRefresh` to TemplateCardProps: `onRefresh?: () => void`
- When status is "ingesting" or "queued", render BELOW the existing card content:
  - If queued: show "Queued for ingestion..." text
  - If ingesting with progress: show `<Progress value={(progress.current / progress.total) * 100} />` and text "Slide {current} of {total}"
  - If ingesting without progress: show indeterminate state "Extracting slides..."
- When status is "failed": show "Ingestion failed" in red text with the existing status badge

**In mastra/index.ts:**

Import at top:
```typescript
import { getDriveClient } from "../lib/google-auth";
import { ingestionQueue } from "../ingestion/ingestion-queue";
```

After the Mastra instance is created (after `export const mastra = new Mastra({...})`), add a self-invoking async function or a `setTimeout(..., 5000)` that starts the polling loop:

**Polling logic (run every 5 minutes = 300_000ms):**
1. Fetch all templates where `accessStatus = "accessible"` AND `ingestionStatus = "idle"` AND `lastIngestedAt IS NOT NULL` (only check already-ingested templates for staleness)
2. For each template (with 200ms delay between Drive API calls):
   a. Call Drive API `files.get({ fileId: template.presentationId, fields: "modifiedTime" })` using `getDriveClient()`
   b. Compare `modifiedTime` with `template.lastIngestedAt`
   c. If `modifiedTime > lastIngestedAt`:
     - Update `sourceModifiedAt` on the template
     - Call `ingestionQueue.enqueue(template.id)` to auto-re-ingest
     - Log: `[staleness] Template "${template.name}" is stale, enqueuing re-ingestion`
3. Wrap each template check in try/catch -- log errors but don't crash the polling loop
4. Use `setInterval` for the 5-minute cycle. Add a console.log on startup: `[staleness] Background polling started (interval: 5m)`

**Important:**
- Do NOT poll templates that are currently being ingested (ingestionStatus != "idle")
- Do NOT poll templates that have never been ingested (lastIngestedAt IS NULL) -- those should be auto-triggered on add
- The first poll should run 10 seconds after startup (not immediately) to let the server finish initializing
- Wrap the entire setInterval setup so it only runs if the environment has Google credentials configured (check env.GOOGLE_CLOUD_PROJECT exists)

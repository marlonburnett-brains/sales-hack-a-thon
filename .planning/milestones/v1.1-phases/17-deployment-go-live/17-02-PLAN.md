<task type="checkpoint:human-action" gate="blocking">
  <name>Task 2: Provision Oracle VM and deploy agent server</name>
  <files>N/A -- external infrastructure provisioning</files>
  <action>
Create the Oracle Cloud VM, install Docker, clone the repo, and deploy the agent server. Claude cannot access the Oracle Cloud console or SSH into the VM.

Part A -- Create the VM:
1. Go to Oracle Cloud Console > Compute > Instances > Create Instance
2. Name: lumenalta-agent (or similar)
3. Image: Canonical Ubuntu 24.04 (aarch64)
4. Shape: VM.Standard.A1.Flex -- 2 OCPUs, 12 GB RAM (within free tier)
5. Boot volume: 50 GB
6. SSH key: Upload or paste your public SSH key
7. Click Create

Part B -- Configure networking:
1. Go to Networking > Virtual Cloud Networks > [your VCN] > Security Lists > Default Security List
2. Add Ingress Rules:
   - Source CIDR: 0.0.0.0/0, Protocol: TCP, Destination Port: 80
   - Source CIDR: 0.0.0.0/0, Protocol: TCP, Destination Port: 443
3. Note the VM's public IP address

Part C -- Configure DNS:
1. Go to your domain registrar's DNS management
2. Create an A record: agent.yourdomain.com -> [VM public IP]
3. Wait for DNS propagation (check with: dig agent.yourdomain.com)

Part D -- SSH into VM and set up Docker:
ssh ubuntu@[VM_PUBLIC_IP]
sudo apt update && sudo apt upgrade -y
sudo apt install -y ca-certificates curl gnupg
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | sudo tee /etc/apt/sources.list.d/docker.list
sudo apt update && sudo apt install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin
sudo usermod -aG docker ubuntu
sudo iptables -I INPUT -p tcp --dport 80 -j ACCEPT
sudo iptables -I INPUT -p tcp --dport 443 -j ACCEPT
sudo apt install -y iptables-persistent
sudo netfilter-persistent save
(Log out and back in for docker group)

Part E -- Clone repo and configure:
sudo mkdir -p /opt/lumenalta && sudo chown ubuntu:ubuntu /opt/lumenalta
cd /opt/lumenalta
git clone https://github.com/[your-org]/lumenalta-hackathon.git .
cp deploy/.env.example .env
nano .env  (Fill in all production values -- see deploy/.env.example for guidance)
chmod 600 .env

Fill in .env with: DATABASE_URL, DIRECT_URL (prod Supabase), GOOGLE_SERVICE_ACCOUNT_KEY, GOOGLE_DRIVE_FOLDER_ID, GOOGLE_TEMPLATE_PRESENTATION_ID, VERTEX_SERVICE_ACCOUNT_KEY (JSON from vertex-service-account.json or same service account), GOOGLE_CLOUD_PROJECT, GOOGLE_CLOUD_LOCATION, AGENT_API_KEY (SAME production key as Vercel), AGENT_DOMAIN=agent.yourdomain.com

Part F -- Build and start:
cd /opt/lumenalta
docker compose build
docker compose up -d
sleep 10
curl -sf http://localhost:4111/health
curl -sf https://agent.yourdomain.com/health

Part G -- Update Vercel AGENT_SERVICE_URL:
Go to Vercel Dashboard > Project Settings > Environment Variables:
Update AGENT_SERVICE_URL (Production) to: https://agent.yourdomain.com
Trigger a redeploy from Vercel Dashboard > Deployments

Part H -- Run database migrations on production:
ssh ubuntu@[VM_IP]
cd /opt/lumenalta
docker compose exec app npx prisma migrate deploy
(Uses prisma migrate deploy, not prisma migrate dev, per project rules)

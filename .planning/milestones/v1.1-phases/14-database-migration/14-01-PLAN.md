  <name>Task 1: Create Supabase dev and prod projects</name>
  <action>Create two Supabase projects and provide connection strings to Claude.</action>
  <instructions>
    1. Go to https://supabase.com/dashboard
    2. Create an organization (e.g., "lumenalta-hackathon")
    3. Create project "lumenalta-dev":
       - Region: US East (N. Virginia / us-east-1)
       - Generate and SAVE the database password (only shown once)
       - Pricing: Free tier
    4. Once project is ready, go to Settings > Database > Connection string:
       - Copy "URI" format (port 5432) -- this is DIRECT_URL
       - Copy "Connection Pooling > URI" format (port 6543) -- this is DATABASE_URL
       - Append `?pgbouncer=true` to the pooled URL (DATABASE_URL)
    5. Create a second project "lumenalta-prod" (same org, same region, same steps)
    6. Provide ALL FOUR connection strings:
       - DEV DATABASE_URL (pooled, port 6543, with ?pgbouncer=true)
       - DEV DIRECT_URL (direct, port 5432)
       - PROD DATABASE_URL (pooled, port 6543, with ?pgbouncer=true)
       - PROD DIRECT_URL (direct, port 5432)

    Format expected:
    ```
    DEV_DATABASE_URL=postgres://postgres.[ref]:[pw]@aws-0-us-east-1.pooler.supabase.com:6543/postgres?pgbouncer=true
    DEV_DIRECT_URL=postgres://postgres.[ref]:[pw]@aws-0-us-east-1.pooler.supabase.com:5432/postgres
    PROD_DATABASE_URL=postgres://postgres.[ref]:[pw]@aws-0-us-east-1.pooler.supabase.com:6543/postgres?pgbouncer=true
    PROD_DIRECT_URL=postgres://postgres.[ref]:[pw]@aws-0-us-east-1.pooler.supabase.com:5432/postgres
    ```

    Note: If Supabase limits free tier to 2 active projects total (across orgs), create both under the same organization.
  </instructions>
  <resume-signal>Paste all four connection strings above</resume-signal>

       - Set `DATABASE_URL` to the dev pooled connection string (port 6543, with `?pgbouncer=true`)
       - Add `DIRECT_URL` set to the dev direct connection string (port 5432)
       - Keep all other env vars unchanged

    2. **Update apps/agent/src/env.ts**:
       - Change `DATABASE_URL` validator from `z.string().min(1)` to `z.string().url()` (Postgres URLs are proper URLs)
       - Add `DIRECT_URL: z.string().url()` to the server object
       - Add a comment: "Supabase direct connection (migrations/introspection)"

    3. **Update apps/agent/prisma/schema.prisma** datasource block:
       ```prisma
       datasource db {
         provider  = "postgresql"
         url       = env("DATABASE_URL")
         directUrl = env("DIRECT_URL")
       }
       ```
       - Change ONLY the datasource block. Do NOT modify the generator block or any model definitions.
       - All 8 models (WorkflowJob, ImageAsset, ContentSource, Company, Deal, InteractionRecord, FeedbackSignal, Transcript, Brief) remain EXACTLY as-is.

    4. **Delete old SQLite migrations entirely**:
       ```bash
       rm -rf apps/agent/prisma/migrations/
       ```

    5. **Generate fresh Postgres baseline migration**:
       ```bash
       cd apps/agent
       mkdir -p prisma/migrations/0_init
       npx prisma migrate diff \
         --from-empty \
         --to-schema-datamodel prisma/schema.prisma \
         --script > prisma/migrations/0_init/migration.sql
       ```

    6. **Inspect the generated SQL** -- verify it contains CREATE TABLE statements for all 8 models with TEXT columns (not JSONB), TEXT ids with cuid defaults, and all indexes/unique constraints.

    7. **Apply the baseline to Supabase dev**:
       ```bash
       cd apps/agent
       npx prisma migrate deploy
       ```

    8. **Regenerate Prisma Client**:
       ```bash
       cd apps/agent
       npx prisma generate
       ```

    IMPORTANT: Do NOT use `prisma db push` (forbidden by CLAUDE.md). Do NOT use `prisma migrate reset`. Use `prisma migrate deploy` to apply the baseline.

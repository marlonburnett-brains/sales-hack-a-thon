Follow the established CLI pattern from `scripts/tts.ts` -- manual process.argv parsing, no external CLI library, clear error messages, summary output.

**render.ts** -- Single-tutorial render:

CLI usage: `pnpm --filter tutorials render <tutorial-name> [--concurrency N]`

1. **parseArgs()**: Extract `tutorialName` (required positional) and `concurrency` (optional, default 2). Print usage and exit(1) if no tutorial name. Validate concurrency is a positive integer.

2. **Pre-validation** (before touching Remotion):
   - Check `audio/{tutorialName}/timing.json` exists. If not: `"Error: timing.json not found at {path}\nRun capture and tts first: pnpm --filter tutorials capture <name> && pnpm --filter tutorials tts <name>"` then exit(1).
   - Check `output/{tutorialName}/` directory exists. If not: `"Error: Screenshot directory not found at {path}\nRun capture first: pnpm --filter tutorials capture <name>"` then exit(1).
   - Load and validate timing manifest with `TimingManifestSchema.parse()`. On Zod error, print the validation error and exit(1).
   - For each step in manifest, check `output/{tutorialName}/{stepId}.png` exists. On missing screenshot, print which file is missing and exit(1).
   - For each step, check `audio/{tutorialName}/{audioFile}` exists. If missing, log a warning but continue (set that step's durationMs to 0 so the composition uses the 3-second fallback).

3. **Bundle**: Use `bundle()` from `@remotion/bundler`:
   - `entryPoint: path.join(process.cwd(), "src/remotion/index.ts")`
   - `publicDir: process.cwd()` -- tutorials app root so staticFile() resolves output/ and audio/

4. **Build inputProps**: Map manifest steps to `StepInput[]` format. For steps with missing audio, set `durationMs: 0` (TutorialComposition handles the 3-second fallback). Pass `tutorialName`, `steps`, and recalculated `totalDurationMs` (sum of actual durationMs values, substituting 3000 for missing audio steps).

5. **Select composition**: `selectComposition({ serveUrl: bundleLocation, id: "tutorial", inputProps })`

6. **Render**: Use `renderMedia()` from `@remotion/renderer`:
   - `codec: "h264"`
   - `crf: 18`
   - `concurrency` from CLI args (default 2)
   - `outputLocation: path.join(process.cwd(), "videos", \`\${tutorialName}.mp4\`)`
   - Create `videos/` directory with `mkdirSync({ recursive: true })` before rendering
   - `onProgress` callback: print progress percentage every 10% (use `\r` for in-place updates)

7. **Summary output** (matching tts.ts pattern):
   ```
   --- Render Summary ---
   Tutorial: {tutorialName}
   Steps: {N}
   Duration: {seconds}s
   Output: videos/{tutorialName}.mp4
   ```

8. **Error handling**: Wrap main() in `.catch()`, print error message, exit(1).

**render-all.ts** -- Batch render:

CLI usage: `pnpm --filter tutorials render:all [--concurrency N]`

1. **Discover tutorials**: Scan `audio/` directory for subdirectories that have both `timing.json` AND a matching `output/{name}/` directory with screenshots. Sort alphabetically.

2. If no tutorials found, print `"No renderable tutorials found. Run capture and tts first."` and exit(0).

3. **Render sequentially**: For each discovered tutorial, import and run the same render logic. Do NOT parallelize -- Remotion docs recommend sequential rendering. Print a separator between tutorials.

4. **Summary**: Print count of successfully rendered tutorials and any failures.

IMPORTANT: Use `@remotion/renderer` programmatic API (bundle + selectComposition + renderMedia), NOT spawning `npx remotion render` as a subprocess. The render script itself should NOT contain any JSX -- JSX lives only in src/remotion/ which webpack bundles.

Import paths for timing manifest: `import { TimingManifestSchema } from "../src/types/timing-manifest.js"` (with .js extension for ESM compatibility, matching tts.ts pattern).

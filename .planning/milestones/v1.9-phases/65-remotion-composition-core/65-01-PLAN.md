Create `apps/tutorials/src/remotion/` directory with four files:

**index.ts** -- Remotion entry point:
- Import `registerRoot` from `remotion`
- Import `RemotionRoot` from `./Root`
- Call `registerRoot(RemotionRoot)`

**Root.tsx** -- Composition registration:
- Export `RemotionRoot` React component
- Register a single `<Composition>` with:
  - `id="tutorial"`
  - `component={TutorialComposition}`
  - `width={1920}`, `height={1080}`, `fps={30}`
  - `durationInFrames={1}` (placeholder -- calculateMetadata overrides)
  - `defaultProps` with `tutorialName: ""`, `steps: []`, `totalDurationMs: 0`
  - `calculateMetadata` that computes `durationInFrames` from `props.totalDurationMs`:
    - `const fps = 30`
    - Calculate total frames as sum of each step's `Math.ceil((step.durationMs / 1000) * fps)` -- this avoids off-by-one gaps (per Research pitfall 4). For steps with `durationMs === 0`, use `Math.ceil((3000 / 1000) * fps)` (3-second fallback per locked decision).
    - Return `{ durationInFrames: Math.max(totalFrames, 1) }`

Define `TutorialProps` type inline in Root.tsx:
```typescript
type StepInput = {
  stepId: string;
  audioFile: string;
  durationMs: number;
};

type TutorialProps = {
  tutorialName: string;
  steps: StepInput[];
  totalDurationMs: number;
};
```

Export `TutorialProps` and `StepInput` types for use by render.ts in Plan 02.

**TutorialComposition.tsx** -- Step-to-Sequence mapping:
- Import `AbsoluteFill`, `Sequence` from `remotion`
- Import `TutorialStep` from `./TutorialStep`
- Import `TutorialProps` from `./Root`
- Constants: `FPS = 30`, `FALLBACK_DURATION_MS = 3000`
- For each step in `props.steps`:
  - Calculate `durationMs = step.durationMs || FALLBACK_DURATION_MS` (missing audio fallback)
  - Calculate `durationInFrames = Math.ceil((durationMs / 1000) * FPS)`
  - Track cumulative `from` offset (starts at 0, incremented by each step's durationInFrames)
  - Render `<Sequence key={step.stepId} from={from} durationInFrames={durationInFrames} name={step.stepId} layout="none">`
    - Use `layout="none"` (NOT the default "absolute-fill") since only one Sequence is active at a time
    - Inside: `<TutorialStep tutorialName={props.tutorialName} stepId={step.stepId} audioFile={step.audioFile} durationMs={durationMs} hasAudio={step.durationMs > 0} />`
- Wrap all Sequences in `<AbsoluteFill style={{ backgroundColor: "#000" }}>`

**TutorialStep.tsx** -- Shared step component (COMP-02):
- Import `AbsoluteFill`, `Img`, `staticFile` from `remotion`
- Import `Audio` from `@remotion/media` (NOT from `remotion` -- per Research recommendation)
- Props: `tutorialName: string`, `stepId: string`, `audioFile: string`, `durationMs: number`, `hasAudio: boolean`
- Screenshot source: `staticFile(\`output/\${tutorialName}/\${stepId}.png\`)`
- Audio source: `staticFile(\`audio/\${tutorialName}/\${audioFile}\`)`
- Render `<AbsoluteFill>` containing:
  - `<Img src={screenshotSrc} style={{ width: "100%", height: "100%", objectFit: "cover" }} />` -- edge-to-edge, no letterboxing (locked decision)
  - `{hasAudio && <Audio src={audioSrc} />}` -- conditional audio, only when audio exists

Do NOT include any visual effects, transitions, overlays, or cursor animations -- those are Phase 66 scope.

**mock-server.ts** -- Add stage state management and sequence support to `createMockServer()`:

1. **State variables** at top of createMockServer: `let currentStage: string = "idle"` and `const sequenceCounters: Record<string, number> = {}`. Load sequences via `loadSequences(tutorialName)` (import from loader.ts).

2. **Sequence helper function** `getNextSequenceResponse(key: string): unknown | null` -- returns `sequences[key][Math.min(idx, seq.length - 1)]`, advances counter, logs `[mock-seq] {key}: response {serving}/{total} (status={status})`. Returns null if no sequence for key.

3. **Control endpoints** (add BEFORE the existing route handlers):
   - `POST /mock/set-stage` -- `{ stage: string }` body, sets `currentStage = stage`, logs `[mock-stage] Stage set to: {stage}`, returns `{ stage }`.
   - `GET /mock/get-stage` -- returns `{ stage: currentStage }` (for browser-side mocks to query).
   - `POST /mock/reset-sequence` -- `{ key: string }` body, sets `sequenceCounters[key] = 0`, logs `[mock-seq] Reset: {key}`, returns `{ key, counter: 0 }`.
   - `POST /mock/reset` -- resets `currentStage = "idle"`, zeros all sequence counters, logs `[mock] Full state reset`, returns `{ stage, sequences: "reset" }`.

4. **Stage-aware route handlers** -- Update these existing handlers to be stage-aware:
   - `GET /deals/:dealId/interactions` -- load stage fixtures via `loadStageFixtures(tutorialName, currentStage)`, if stage fixtures have `interactions`, deep-merge onto base fixtures, return result. Import `deepMerge` and `loadStageFixtures` from loader.ts.
   - `GET /briefs/:briefId` and `GET /briefs/:briefId/review` -- if stage fixtures have brief overrides, merge them.
   - Workflow run endpoints (`GET /api/workflows/*/runs/:runId`) -- check for sequence response first via `getNextSequenceResponse("workflow-status")`, if available return it; otherwise derive from currentStage: idle/completed->"completed", generating->"running", skeleton/lowfi/hifi->"suspended".
   - `GET /templates/:id/progress` -- check for sequence via `getNextSequenceResponse("ingestion-progress")`, if available return it; otherwise return `{ status: "idle", current: 0, total: 0 }`.

5. **Exclude control routes from logging** -- update the request logging middleware to also skip `/mock/` prefixed routes (like it already skips `/auth/`).

- New optional third parameter: `options?: { stageGetter?: () => string; sequenceGetter?: (key: string) => unknown | null }`

Fallback to existing "completed" response if no options provided.

- Add pre-step sequence reset: if `step.resetSequences`, loop and call `fetch(http://localhost:4112/mock/reset-sequence, ...)` for each key.
- Add post-action `waitForText`: if `step.waitForText`, call `await waitForText(page, step.waitForText)` (import from determinism.ts).
- Add post-action delay: if `step.delayMs`, call `await page.waitForTimeout(step.delayMs)`.
- Create shared mutable stage ref: `let currentStageRef = "idle"`. Update it when `step.mockStage` is set. Pass `{ stageGetter: () => currentStageRef, sequenceGetter: ... }` to `mockBrowserAPIs()`.

The simpler approach: pass `stageGetter` that fetches from mock server synchronously via the mutable ref (updated in the main loop), and skip sequenceGetter for now since workflow status polling in browser goes through the stage-derived approach. Only mock server handles sequence-based responses for SSR routes.

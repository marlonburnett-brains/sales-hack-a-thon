  <files>apps/tutorials/scripts/capture.ts, apps/tutorials/src/helpers/screenshot.ts</files>
  <action>
1. Create `apps/tutorials/src/helpers/screenshot.ts`:
   - Export `getScreenshotPath(tutorialId: string, stepIndex: number)`: Returns path like `output/getting-started/step-001.png` with zero-padded step numbers (3 digits)
   - Export `captureStep(page: Page, tutorialId: string, stepIndex: number, options?: { fullPage?: boolean })`:
     a. Call prepareForScreenshot(page) from determinism helper
     b. Calculate output path using getScreenshotPath
     c. Ensure output directory exists (mkdirSync recursive)
     d. Call page.screenshot({ path, fullPage: options?.fullPage ?? false, type: 'png' })
     e. Log: `  [step-{NNN}] Captured: {path}`
     f. Return the path for reference
   - Export `ensureOutputDir(tutorialId: string)`: Creates output/{tutorialId}/ directory

2. Create `apps/tutorials/scripts/capture.ts`:
   - Parse CLI args: first positional arg is tutorial name (e.g., "getting-started")
   - If no tutorial name provided, print usage and exit
   - Load the tutorial script from `fixtures/{tutorialName}/script.json` using TutorialScriptSchema.parse() for validation
   - Start the mock server: `await startMockServer(tutorialName, 4112)`
   - Run Playwright via child_process.execSync or spawn:
     ```
     npx playwright test capture/{tutorialName}.spec.ts --project=capture
     ```
   - Pass environment variable TUTORIAL_NAME={tutorialName} to Playwright
   - After Playwright completes (success or failure), shut down mock server
   - Print summary: number of screenshots captured, output directory path
   - Exit with Playwright's exit code

3. Wire the package.json "capture" script:
   - Should already be "tsx scripts/capture.ts" from Plan 01
   - Verify it works as: `pnpm --filter tutorials capture getting-started`

4. Handle errors gracefully:
   - If script.json doesn't exist, print helpful error message about generating it first
   - If mock server fails to start, print port conflict hint
   - Always shut down mock server in finally block
  </action>

In `apps/tutorials/src/remotion/Root.tsx`:
- Extend `StepInput` with the optional fields introduced in Plan 01: zoomTarget, callout, shortcutKey, cursorTarget, cursorFrom, hasCursorAction, stepIndex, totalSteps.
- Extend `TutorialProps` with `title`, `description`, and optional `nextTutorialName`.
- Add constants for:
  - `FPS = 30`
  - `FALLBACK_DURATION_MS = 3000`
  - `INTRO_DURATION_FRAMES = 90`
  - `OUTRO_DURATION_FRAMES = 120`
  - `TRANSITION_DURATION_FRAMES = 15`
- Update `calculateMetadata` to account for intro, outro, and transition overlap:
  - Sum all step frame durations using fallback logic for missing audio.
  - Add intro and outro durations.
  - Subtract transition overlap for every cut point.
  - For a tutorial with `N` steps and both bookends present, subtract `(N + 1) * TRANSITION_DURATION_FRAMES`.
  - Clamp to at least 1 frame.

In `apps/tutorials/scripts/render.ts`:
- Keep all existing pre-validation.
- In addition to timing.json, load and validate the tutorial script JSON so render-time props can include title, description, zoom, callout, shortcut, and cursor information.
- Merge script step metadata with timing manifest steps by `step.id` / `stepId`.
- Build `StepInput[]` with:
  - audio and duration data from timing manifest
  - visual metadata from script
  - `stepIndex` and `totalSteps`
  - `hasCursorAction` derived strictly from click or hover script actions
  - `cursorFrom` computed from the previous click or hover step so cursor motion has continuity between steps
- Treat `cursorTarget` only as coordinate data for click or hover steps. It must not independently cause the cursor to appear on informational steps.
- Pass `title`, `description`, `nextTutorialName` (if derivable or available; otherwise omit), and steps in inputProps.

For script discovery:
- Prefer the established fixture script locations already in the repo, such as `fixtures/{tutorialName}/script.json`.
- Fail clearly if timing.json exists but the matching script cannot be found.

In `apps/tutorials/scripts/render-all.ts`:
- Keep sequential rendering.
- Reuse the exported single-tutorial render path without duplicating logic.
- Ensure any new script dependency assumptions still allow render-all discovery to function predictably.

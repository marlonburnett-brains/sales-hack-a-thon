   - Compute VENV_DIR as `$(cd "$(dirname "$0")/.." && pwd)/.venv`
   - Check python3.11 exists, if not print: "Python 3.11 required. Install with: brew install python@3.11"
   - Create venv: `python3.11 -m venv "$VENV_DIR"`
   - Activate: `source "$VENV_DIR/bin/activate"`

2. **Create Python generation script** at `scripts/chatterbox-generate.py`:
   - Shebang: `#!/usr/bin/env python3`
   - Docstring: "Generate a single WAV file from text using Chatterbox-Turbo."
   - Use argparse with: --text (required), --output (required), --reference (required), --emotion (optional)
   - Device detection: `device = "mps" if torch.backends.mps.is_available() else "cpu"`
   - CRITICAL: Load model to CPU first: `model = ChatterboxTurboTTS.from_pretrained("cpu")`
   - Then move components to target device if not CPU:
     ```python
     if device != "cpu":
         if hasattr(model, "t3"): model.t3 = model.t3.to(device)
         if hasattr(model, "s3gen"): model.s3gen = model.s3gen.to(device)
         if hasattr(model, "ve"): model.ve = model.ve.to(device)
     ```
   - If emotion provided, prepend as paralinguistic tag: `text = f"[{args.emotion}] {text}"`
   - Generate: `wav = model.generate(text, audio_prompt_path=args.reference)`
   - Save: `torchaudio.save(args.output, wav, model.sr)`
   - Import: `from chatterbox.tts_turbo import ChatterboxTurboTTS` (inside main to avoid slow import at module level)

3. **Update .gitignore** to add `.venv/` and `audio/` entries:
   - Read existing .gitignore first

1. **Create ChatterboxEngine** at `src/tts/chatterbox-engine.ts`:
   - Import TTSEngine from `./engine.ts` and getWavDurationMs from `./wav-utils.ts`
   - Import `spawn` from `node:child_process`, `fs` from `node:fs`, `path` from `node:path`
   - Implements TTSEngine interface
   - Constructor takes `private referenceAudioPath: string` (path to reference voice clip)
   - `generate()` method:
     a. Compute venv Python path: `path.join(process.cwd(), ".venv", "bin", "python")`
     b. Check if venv Python exists (`fs.existsSync`). If not, throw Error:
        "Chatterbox venv not found. Run: bash scripts/setup-chatterbox.sh"
        Do NOT silently fall back to Kokoro -- this is a locked user decision.
     c. Check if reference audio exists. If not, throw Error with instructions.
     d. Build args array: `["scripts/chatterbox-generate.py", "--text", text, "--output", outputPath, "--reference", this.referenceAudioPath]`
     e. If `options?.emotion` provided, push `"--emotion"` and the emotion value to args
     f. Spawn the Python process using a Promise wrapper (same pattern as capture.ts):
        ```typescript
        await new Promise<void>((resolve, reject) => {
          const proc = spawn(venvPython, args, {
            cwd: process.cwd(),
            stdio: ["ignore", "pipe", "pipe"],
          });
          let stderr = "";
          proc.stderr?.on("data", (d: Buffer) => { stderr += d.toString(); });
          proc.on("close", (code) => {
            if (code !== 0) reject(new Error(`Chatterbox failed (exit ${code}): ${stderr}`));
            else resolve();
          });
        });
        ```
     g. Return `{ durationMs: getWavDurationMs(outputPath) }`
  </action>

4. **Create WAV utilities** at `src/tts/wav-utils.ts`:
   - `getWavDurationMs(filePath: string): number` -- reads 44-byte WAV header to extract sample rate, bits per sample, channels, data chunk size, then calculates duration in ms
   - Do NOT load the entire file -- only read the 44-byte header using `fs.openSync`/`fs.readSync`/`fs.closeSync`
   - Parse: sampleRate at offset 24 (UInt32LE), bitsPerSample at offset 34 (UInt16LE), numChannels at offset 22 (UInt16LE), dataSize at offset 40 (UInt32LE)
   - Formula: `Math.round((dataSize / (numChannels * (bitsPerSample / 8))) / sampleRate * 1000)`

2. **Create KokoroEngine** at `src/tts/kokoro-engine.ts`:
   - Implements TTSEngine interface from `./engine.ts`
   - Lazy-loads the model: store `private tts: any | null = null` (kokoro-js types may not be available)
   - On first `generate()` call, dynamically import kokoro-js: `const { KokoroTTS } = await import("kokoro-js")`
   - Load model: `KokoroTTS.from_pretrained("onnx-community/Kokoro-82M-v1.0-ONNX", { dtype: "q8", device: "cpu" })`
   - Log "Downloading Kokoro model (~160MB, first run only)..." before `from_pretrained` call to prevent confusion on first run
   - Voice preset: try `af_heart` (default warm female). Set as a constant at top of file so it's easy to change.
   - IMPORTANT: Strip emotion tags from narration text before passing to Kokoro. Kokoro has NO paralinguistic tag support -- it would speak "[cheerful]" literally. Simply remove the emotion option, don't prepend anything.
   - Call `audio.save(outputPath)` to write the WAV file
   - Return `{ durationMs }` using `getWavDurationMs` from `./wav-utils.ts` on the saved file

3. **Create post-processing module** at `src/tts/post-process.ts`:
   - Export `async function postProcessAudio(inputPath: string, outputPath: string): Promise<void>`
   - First check ffmpeg is available: spawn `ffmpeg -version`, if fails throw Error with message "ffmpeg not found. Install with: brew install ffmpeg"
   - Use two-pass ffmpeg loudnorm:
     - Pass 1: `ffmpeg -i {input} -af loudnorm=I=-16:TP=-1.5:LRA=11:print_format=json -f null -` -- parse JSON output from stderr to get measured_I, measured_LRA, measured_TP, measured_thresh
     - Pass 2: `ffmpeg -y -i {input} -af "loudnorm=I=-16:TP=-1.5:LRA=11:measured_I={measured_I}:measured_LRA={measured_LRA}:measured_TP={measured_TP}:measured_thresh={measured_thresh},apad=pad_dur=0.5" -ar 24000 -ac 1 {output}`
   - Use a temp file for intermediate output if input and output paths are the same
   - Use spawn from child_process with Promise wrapper (match capture.ts pattern)
   - Collect stderr to parse the loudnorm JSON from pass 1

4. **Create manifest module** at `src/tts/manifest.ts`:
   - Import TimingManifestSchema, TimingEntrySchema from `../types/timing-manifest.ts`
   - `readManifest(manifestPath: string): TimingManifest | null` -- reads and validates, returns null if file doesn't exist
   - `writeManifest(manifestPath: string, manifest: TimingManifest): void` -- Zod parse then JSON.stringify with 2-space indent
   - `updateManifestEntry(manifestPath: string, stepId: string, entry: TimingEntry, engine: "kokoro" | "chatterbox"): void` -- reads existing manifest (or creates empty), finds entry by stepId and replaces (or appends), recalculates totalDurationMs as sum of all step durations, updates generatedAt timestamp, writes back
   - Ensure output directory exists (mkdirSync recursive) before writing
